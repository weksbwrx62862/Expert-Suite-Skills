# /竞品追踪 -- 竞品动态与定位分析

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

追踪竞品动态，进行结构化对比分析，输出品牌定位深度对比、数字化声量监测、营销策略逆向工程、功能矩阵、SWOT分析、销售战斗卡片和差异化竞争策略。

## 能力概览

### 基础能力（独立可用）
- 使用 agent 搜索能力实时检索竞品新闻、产品更新、融资动态
- 品牌定位深度分析（定位声明逆向工程 + 叙事分析）
- 功能对比矩阵 + 定价策略对比 + 竞争定位矩阵
- 数字化竞品声量对比（百度指数、微信指数、抖音搜索量）
- SWOT 分析 + 交叉策略（SO/ST/WO/WT）
- 竞品营销策略逆向工程（投放渠道、内容策略、活动节奏）
- 销售战斗卡片（Battlecard）生成
- 市场信号解读（从竞品动作推断战略意图）
- 分层差异化建议（短期 / 中期 / 长期）

### 增强能力（连接 MCP 后解锁）
- 连接 ~~Notion → 竞品报告、功能矩阵和动态时间线写入 Notion，支持长期动态监测

## 连接器（可选增强）

| 连接器 | 增强能力 |
|--------|---------|
| **~~Notion**（Notion） | 竞品报告和功能矩阵写入 Notion，便于管理层查阅和长期追踪 |

> 没有连接器也完全可以使用——agent 自带搜索能力，可实时检索竞品公开信息。

## 调用方式

```
/竞品追踪 <竞品名称>
```

追踪竞品：@$1

## 输入要求

**必填**：
1. **竞品名称**：1-5 个竞品公司或产品名称

**选填**：
2. **分析维度**：产品功能 / 定价策略 / 品牌定位 / 数字化声量 / 营销策略 / 市场表现 / 融资动态 / 全面分析（默认：全面分析）
3. **我方信息**：自身产品或公司基本情况（用于对比，未提供则只做竞品画像）
4. **关注时间范围**：近1周 / 近1月 / 近3月 / 近1年（默认：近3月）
5. **分析模式**：快速扫描 / 深度对标 / 融资跟踪 / 定期追踪（默认：深度对标）
6. **输出需求**：是否需要销售战斗卡片（Battlecard）

## 执行流程

### 第一步：信息收集

使用 agent 自带搜索能力，按优先级检索竞品信息：

**一级来源（直接信息）**：
1. 竞品官网（首页定位、产品页、定价页、关于页、招聘页）
2. 竞品官方社媒（公众号、抖音号、小红书企业号）
3. 竞品新闻稿和公告

**二级来源（第三方验证）**：
4. 行业媒体报道（36Kr、虎嗅、钛媒体等）
5. 应用市场评价（App Store / 应用宝 / 酷安）
6. 用户论坛和社区口碑（知乎、脉脉、小红书）
7. 招聘平台（拉勾、BOSS直聘 — 招聘方向反映战略重心）

**交叉验证规则**：关键事实（融资金额、产品功能）须至少2个独立来源确认。

### 第二步：品牌定位深度分析

#### 2.1 定位声明逆向工程

为每个竞品还原其定位声明：

> 对于 [目标人群]，[产品名] 是 [品类] 中能 [核心差异化价值] 的选择，因为 [信任理由]。

#### 2.2 叙事分析框架

解构竞品品牌叙事的核心元素：

| 叙事元素 | 说明 | 竞品A | 竞品B |
|---------|------|------|------|
| **反派** | 品牌对抗的敌人（旧方案/复杂性/高成本） | [描述] | [描述] |
| **英雄** | 叙事中的主角（客户/产品/团队） | [描述] | [描述] |
| **变革** | 承诺的前后变化 | [描述] | [描述] |
| **后果** | 不行动的代价 | [描述] | [描述] |

> 叙事分析揭示竞品的情感营销策略和差异化定位逻辑。

#### 2.3 品牌定位画像

| 维度 | 竞品A | 竞品B | 我方 |
|-----|------|------|------|
| **目标人群** | [核心用户群体画像] | ... | ... |
| **价值主张** | [核心承诺/品牌口号] | ... | ... |
| **品牌调性** | [如：专业/年轻/温暖] | ... | ... |
| **情感诉求** | [解决什么情感需求] | ... | ... |
| **定价心智** | [高端/性价比/免费增值] | ... | ... |

#### 2.4 品类策略判断

判断每个竞品采用的品类策略：
- **开创新品类**：定义全新赛道（高风险高回报）
- **重新定义品类**：改变评价标准使自身优势突出
- **赢得既有品类**：在公认标准上超越竞品
- **切分细分市场**：在细分场景或人群中建立优势

|-----|------|------|------|------|

| 抖音搜索量（周均） | [数值] | [数值] | [数值] | [趋势] |

#### 3.2 SEO 关键词竞争分析

| 关键词 | 竞品A 排名 | 竞品B 排名 | 我方排名 | 竞争机会 |
|-------|-----------|-----------|---------|---------|
| [行业核心词] | [排名] | [排名] | [排名] | [高/中/低] |
| [产品功能词] | [排名] | [排名] | [排名] | [机会评估] |

### 第四步：结构化产品分析

#### 4.1 产品功能矩阵

| 功能维度 | 我方 | 竞品A | 竞品B | 差距判断 |
|---------|-----|------|------|---------|
| [核心功能1] | [具体描述] | [具体描述] | [具体描述] | [领先/持平/落后] |

> 填充规则：每个单元格不只写"有/无"，还要简述具体实现水平和用户体验差异。

#### 4.2 定价策略对比

| 维度 | 我方 | 竞品A | 竞品B |
|-----|-----|------|------|
| 定价模式 | [订阅/买断/免费增值] | ... | ... |
| 价格区间 | [具体价格] | ... | ... |
| 免费版限制 | [具体限制] | ... | ... |
| 付费转化策略 | [策略描述] | ... | ... |

#### 4.3 竞争定位矩阵

用文字描述二维定位（横轴：价格高→低，纵轴：功能简→全）：
- 说明各竞品在矩阵中的位置
- 标注市场空白区域（潜在机会）
- 建议我方应占据的位置及理由

### 第五步：竞品营销策略逆向工程

#### 5.1 投放渠道分析

| 竞品 | 主力投放渠道 | 投放形式 | 预估月投放量级 | 投放策略特点 |
|-----|------------|---------|-------------|------------|
| 竞品A | [渠道] | [形式] | [估算] | [策略特点] |

#### 5.2 内容策略分析

| 维度 | 竞品A | 竞品B |
|-----|------|------|
| 内容主阵地 | [平台+频率] | ... |
| 内容类型偏好 | [种草/教程/测评/品牌故事] | ... |
| 核心内容主题 | [TOP3话题] | ... |
| 用户互动策略 | [评论区/私信/社群] | ... |

#### 5.3 消息传递优劣势评估

对每个竞品的营销信息评估：
| 维度 | 评分 | 说明 |
|-----|------|------|
| 清晰度 | [高/中/低] | 首次访客5秒内能否理解产品 |
| 差异性 | [高/中/低] | 定位是否独特还是泛泛而谈 |
| 可信度 | [高/中/低] | 是否用证据支撑宣称 |
| 一致性 | [高/中/低] | 跨渠道信息是否一致 |
| 共鸣度 | [高/中/低] | 是否击中真实用户痛点 |

### 第六步：SWOT 分析（相对于主要竞品）

- **S-优势**：我方相对竞品的明确优势点（具体到功能/数据）
- **W-劣势**：我方相对竞品的明确短板
- **O-机会**：竞品未覆盖的市场空白或其犯的错误
- **T-威胁**：竞品的强势动作或市场变化对我方的影响

交叉策略：
- **SO策略**：利用优势抓住机会（进攻型）
- **WO策略**：克服劣势抓住机会（补强型）
- **ST策略**：利用优势应对威胁（防御型）
- **WT策略**：规避劣势和威胁（收缩型）

### 第七步：销售战斗卡片（Battlecard）

如用户需要，为主要竞品生成可直接用于销售团队的一页参考：

| 模块 | 内容 |
|-----|------|
| **竞品一句话** | 他们做什么 + 面向谁 |
| **他们的话术** | 主Slogan + TOP3宣称 |
| **他们的强项** | 客观承认的优势（诚实） |
| **他们的弱项** | 可验证的短板和用户差评主题 |
| **我方差异化** | 3-5个具体差异点（差异+为何重要+证据） |
| **常见异议应对** | "他们更便宜"→回应 / "他们功能更多"→回应 |
| **埋雷问题** | 引导客户问竞品的问题（暴露其弱点） |
| **拆雷问题** | 竞品可能引导客户问我们的问题 + 应对 |

### 第八步：市场信号解读与差异化建议

#### 市场信号解读

| 竞品动作 | 可能的战略意图 | 对我方的影响 | 建议应对 |
|---------|-------------|------------|---------|
| [如：推出免费版] | [如：抢占下沉市场] | [如：低端用户流失] | [如：差异化高端定位] |
| [如：大规模招聘AI工程师] | [如：押注AI方向] | [如：技术代差风险] | [如：加速自研或合作] |

#### 差异化建议

| 建议 | 时间框架 | 影响力 | 可行性 | 优先级 |
|-----|---------|-------|-------|-------|
| [具体建议1] | 短期1-3月 | 高/中/低 | 高/中/低 | [P0/P1/P2] |
| [具体建议2] | 中期3-6月 | 高/中/低 | 高/中/低 | [P0/P1/P2] |

#### 定位避坑提醒
- 避免「对着竞品定位」而非「对着用户需求定位」
- 避免宣称过多差异化点（选最重要的1-2个）
- 避免使用用户不用的品类术语
- 避免基于功能定位而非成果定位
- 避免频繁变更定位（市场认知需要时间建立）

### 第九步：验证与标注

- 所有事实性信息标注数据来源（"来源：XX报道 / 官网 / 公开财报"）
- 推测性判断标注"[推测]"标签
- 缺乏信息的维度标注"[信息不足]"而非编造
- 价格/融资等敏感数据注明获取时间和来源
- 声量数据标注采集时间点

## 质量标准

- 所有数据必须标注来源和可信度等级
- 推测性判断必须明确标注，不得混淆为事实
- 差异化建议必须具体可执行，不能是"加强产品力"这种空话
- 信息不足的维度诚实标注，不编造数据
- 竞品信息以公开来源为准，不涉及商业机密
- 营销策略逆向工程基于可观测信号，不过度推测
- Battlecard 内容须可直接供销售团队使用，语言简洁直接

## 场景变体

### 快速扫描 — 用户只需了解竞品最新动态，跳过 SWOT 和定价分析，直接输出动态时间线 + 简要判断
### 深度对标 — 完整执行所有分析模块，适合季度/年度竞品报告
### 融资跟踪 — 聚焦竞品融资动态、估值变化和资本市场表现
### 定期追踪 — 输出可复用的追踪模板，标注需要每周/月更新的关键指标
### 战斗卡片 — 聚焦生成销售可用的竞品 Battlecard，适合销售赋能场景

## 关联技能

- **营销文案** — 基于竞品差异点撰写差异化营销内容
- **活动策划** — 了解竞品市场动作后设计针对性活动方案
- **社媒热点追踪** — 竞品相关舆情也是值得关注的市场信号
- **营销效果分析** — 与竞品投放效果对标，验证自身投放效率
- **SEO内容优化** — 基于竞品关键词竞争分析，优化自身 SEO 策略
- **品牌调性审查** — 确保差异化信息与品牌调性一致
