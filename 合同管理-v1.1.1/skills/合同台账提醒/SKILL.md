# /合同台账提醒 -- 合同日期管理与到期提醒

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

管理合同关键日期节点，自动检测自动续约条款，生成分级到期提醒和履约节点追踪。

**独立能力（无需连接器）**

- 合同日期提取与倒计时计算
- 自动续约条款检测与提醒
- 三级紧急度分类（红/黄/绿）
- 履约节点追踪（付款/交付/验收）
- 输出提醒清单或 iCal 格式文件

**增强能力（连接器加持）**

- ~~Notion -> 将台账报告发布到 Notion，便于团队共享和持续跟踪

## 连接器（可选增强）

| 连接器 | 增强能力 |
|-----------|---------|
| **~~Notion** | 将台账报告、提醒清单发布到 Notion，便于团队查阅和持续更新 |

> 没有连接器也完全可以使用——输出提醒清单表格或 iCal 文件供手动导入。

## 调用方式

```
/合同台账提醒 <合同信息>
```

管理合同台账：@$1

## 输入要求

接受以下形式的输入：

**方式一：直接描述**
提供合同名称、签约双方、关键日期即可。

**方式二：上传合同文件**
上传合同PDF/DOCX，自动提取关键日期条款。

**方式三：批量录入**
上传Excel台账表，包含以下字段：

| 字段 | 必填 | 说明 |
|------|------|------|
| 合同名称 | 是 | 合同标题或编号 |
| 签约双方 | 是 | 甲方、乙方名称 |
| 签约日期 | 是 | 合同签署日期 |
| 生效日期 | 否 | 默认等于签约日期 |
| 到期日期 | 是 | 合同终止日期 |
| 续签截止日 | 否 | 需在此日前决定是否续签 |
| 履约节点 | 否 | 付款/交付/验收等里程碑日期 |
| 提醒天数 | 否 | 提前提醒天数（默认30天） |
| 负责人 | 否 | 合同对接人 |

## 执行流程

### 第一步：解析合同日期

**如输入为合同文件：**
自动提取以下关键日期条款：
- 合同期限条款（生效日期、到期日期）
- 付款条款中的付款节点
- 交付/验收条款中的时间节点
- 质保期/维保期条款
- 续约/续展条款

**如输入为文本或Excel：**
解析用户提供的日期数据，执行数据校验：
- 到期日期不能早于生效日期
- 续签截止日不能晚于到期日期
- 日期格式不一致时统一转换并请用户确认
- 标记缺失字段而非跳过

### 第二步：自动续约条款检测

检查合同中是否存在自动续约/自动续展条款，这是合同管理中最高优先级的风险点。

**常见自动续约模式及应对**：

| 续约模式 | 典型条款表述 | 风险等级 | 应对策略 |
|---------|------------|---------|---------|
| **自动续展** | "期满后自动续展X年，除非提前X天书面通知不续展" | 高 | 必须在通知截止日前决策，否则自动绑定 |
| **沉默续约** | "期满后任何一方未提出异议则视为续约" | 高 | 主动标记决策节点 |
| **逐年续展** | "每年自动续展一年" | 中 | 年度审查是否仍需续约 |
| **无自动续约** | "期满即终止" | 低 | 如需续约，提前启动谈判 |

**自动续约条款的司法效力说明**：
- 自动续展条款一般有效，未在约定期限内通知不续展的，合同继续有效
- 沉默续约需注意：《民法典》第734条规定租赁合同期满承租人继续使用且出租人未异议的，原合同继续有效但转为不定期
- 建议：对金额较大的合同，即使有自动续约条款，仍应在决策截止日前主动审查

对于存在自动续约条款的合同，计算关键日期：
- **决策截止日** = 到期日期 - 提前通知期
- **提醒日** = 决策截止日 - 提前提醒天数

### 第三步：生成分级提醒计划

计算每个日期节点距今天的剩余天数，按紧急程度分类：

| 级别 | 条件 | 标记 | 建议动作 |
|------|------|------|---------|
| **紧急** | 7天内到期 | 红色 | 立即处理：确认续签/终止/付款/交付 |
| **关注** | 8-30天 | 黄色 | 开始准备：启动续签谈判/备货/预算审批 |
| **正常** | 31-90天 | 绿色 | 纳入计划：列入月度复查清单 |
| **远期** | >90天 | 灰色 | 按计划跟踪 |

**法定节假日提前提醒规则**：
- 如截止日落在法定节假日期间（春节、国庆等），自动提前至节前最后一个工作日
- 春节前15天、国庆前7天额外触发一次提醒
- 跨年合同注意会计年度结算影响（12月中旬额外提醒）

**合同生命周期状态标记**：

| 状态 | 条件 | 含义 |
|------|------|------|
| 履行中 | 未到期且无异常 | 正常履行 |
| 即将到期 | 30天内到期 | 需决策续签或终止 |
| 待续签 | 已决定续签但未完成 | 跟踪续签进度 |
| 已到期 | 超过到期日 | 确认是否已续签或终止 |
| 已终止 | 主动终止或到期未续 | 检查终止后义务（保密、质保等） |

### 第四步：创建提醒

**如已连接 ~~Notion：**
1. 将台账报告发布到 Notion，包含：合同名称、状态、到期日期、剩余天数、续约模式、负责人
2. 便于团队查阅和持续更新

**如未连接 ~~Notion：**
1. 生成结构化的提醒清单表格
2. 输出 iCal (.ics) 格式文件，用户可导入任意日历应用
3. 提供Excel格式的台账表，可用于后续跟踪

## 质量标准

- 日期计算必须准确，标注今天日期作为计算基准
- 自动续约条款必须主动检测和预警，这是最高优先级检查项
- 续签提醒需考虑合同约定的提前通知期（通常30-90天）
- 法定节假日前的到期合同须自动提前提醒
- 批量导入需验证数据完整性，标记缺失字段而非跳过
- 存在歧义的日期表述（如"合同签署后30个工作日"）需标注并请用户确认具体日期
- 不遗漏隐藏在合同条款中的时间节点（如"付款后60天内提供发票"）

## 关联技能

- **起草合同** -- 起草续签合同或补充协议
