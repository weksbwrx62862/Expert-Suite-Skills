# /起草合同 -- 专业级合同初稿生成

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

根据用户描述的交易背景，生成符合中国法律要求、内含行业最佳实践的专业级合同初稿。

**重要提示**：本技能辅助法律工作流程，不构成法律意见。生成的合同初稿应经执业律师审核后方可使用。

**独立能力（无需连接器）**

- 7大合同类型深度支持（采购/SaaS/技术开发/租赁/股权转让/劳务外包/代理经销）
- 200+关键法条精准映射（《民法典》合同编+典型合同分编+司法解释）
- 商业参数以行业基准值填充（付款比例、违约金比率、质保期等）
- 内置"防弹"条款设计（举证分配、通知送达、解释规则锁定）
- 财税条款一体化（发票类型、税费承担、价格调整机制）
- 输出排版规范的 Word 文档

**增强能力（连接器加持）**

- ~~Notion -> 将合同初稿直接发布到 Notion，便于团队协作审阅

```

```

接受文字描述的交易背景，也可上传参考合同或条款要求。

## 输入要求

1. **合同类型**：采购合同 / SaaS服务 / 技术开发 / 租赁合同 / 股权转让 / 劳务外包 / 代理经销 / 其他
2. **签约双方**：甲乙方名称及角色（如"我方为采购方"）
3. **核心交易内容**：标的物/服务内容、金额/价格、期限
4. **特别关注**（可选）：特殊条款要求、我方核心诉求、风险偏好

用户只提供部分信息也可开始，其余采用行业惯例填充并用【待确认：XXX】标注。

## 执行流程

### 第一步：信息采集与合同定型

**1.1 确认合同类型，识别我方立场**

必须明确：合同类型 + 我方角色（买方/卖方、委托方/受托方、出租方/承租方等）。我方立场决定条款倾向性设计——除非用户明确要求"平衡型"起草，否则默认保护我方利益。

**1.2 签约方信息采集**

合同首部对方信息用【待确认：对方公司全称/统一社会信用代码/法定代表人】标注。如用户提供了对方工商信息，直接填充至合同首部。

**1.3 加载合同类型专属框架**

所有合同均包含"通用底座条款"+"类型专属条款"两层结构。

**通用底座条款（所有合同必备14项）：**

| 序号 | 条款模块 | 核心内容 | 关键法律依据 |
|-----|---------|---------|-------------|
| 1 | 合同首部 | 双方基本信息（全称+信用代码+地址+法定代表人+联系方式） | 第469条 |
| 2 | 定义与解释 | 关键术语精确定义，消除歧义 | 第466条 |
| 3 | 标的与范围 | 内容明确描述，附技术规格书/需求说明书 | 第470条 |
| 4 | 价款与支付 | 含税/不含税明示、付款节点、触发机制、逾期违约金 | 第511条 |
| 5 | 发票与税务 | 发票类型、开票时间、税率变更处理 | 《增值税法》 |
| 6 | 双方权利义务 | 按我方立场设计，义务具体到可执行动作 | 第509条 |
| 7 | 违约责任 | 阶梯式违约金、违约金与赔偿并行、根本违约定义 | 第577-585条 |
| 8 | 保密条款 | 保密范围（列举+兜底）、期限2-5年、例外、违约救济 | 《反不正当竞争法》第9条 |
| 9 | 不可抗力 | 扩展定义、通知义务（14日内）、持续超60日的解除权 | 第590条 |
| 10 | 合同变更与解除 | 变更须书面、约定解除条件清单 | 第543条、第562-564条 |
| 11 | 争议解决 | 仲裁或诉讼二选一，具体到机构全称或法院 | 《民事诉讼法》第35条 |
| 12 | 通知送达 | 确认地址、送达方式效力、变更通知义务 | 司法实务最佳实践 |
| 13 | 其他约定 | 完整协议、弃权条款、可分割性、附件效力优先顺序 | 第466条、第507条 |
| 14 | 签章页 | 单位名称、法代/授权代表、日期、盖章位置 | 第490条 |

#### 【采购/买卖合同】

| 条款 | 起草要点 | 行业基准参数 |
|-----|---------|------------|
| 质量标准 | 引用国标/行标编号，附《技术规格书》，约定"标准冲突时就高适用" | 优先级：样品>企业标准>行标>国标 |
| 验收程序 | 三阶段：初验(到货3-7日) → 终验(安装后7-30日) → 默示验收 | 默示验收期：普通货物7天，设备类15-30天 |
| 质保期 | 自验收合格之日起算，约定质保范围和响应时间 | 通用设备12-24月，IT 1-3年，消耗品3-6月 |
| 风险转移 | 明确"交付"定义（签收单/验收单），约定运输保险 | 第604条 |
| 退换货 | 按类型分列：质量不合格(无条件)、规格不符(7天)、买方原因(5%-15%手续费) | 通知期限5-10个工作日 |

**付款基准**：预付30% + 验收65% + 质保金5%（质保期满付清），收到专票后15个工作日内。

**关键法条**：质量标准第615-617条、检验第620-624条、风险转移第604-607条、所有权保留第641-643条。

**高频争议防范**：验收标准模糊→附详细技术参数表；默示验收陷阱→区分"签收"与"验收"；逾期交付→阶梯违约金（1-15日万分之三/日，16-30日万分之五/日，超30日可解除）。

#### 【SaaS/软件服务合同】

| 条款 | 起草要点 | 行业基准参数 |
|-----|---------|------------|
| SLA服务等级 | 明确可用性、计算口径、排除项，约定监测依据 | 99.9%可用性=年停机≤8.76h，低于99%赔偿30% |
| 数据安全 | 服务商为"受托处理者"，约定目的限制、子处理者限制、泄露通知时限 | 泄露后2h内通知，日志保存≥6月 |
| 数据归属 | 客户业务数据归客户所有，衍生数据须明确约定 | 客户输入=客户，系统日志=服务商 |
| 退出迁移 | 过渡期30-90天、标准格式导出(CSV/JSON/API)、数据删除+删除证明 | 基础导出免费，人工协助2000-5000元/人天 |
| 续费与价格 | 自动续约通知期30-60天、价格锁定期、涨幅上限 | 首签锁2-3年，年涨幅≤5%-10% |

**数据合规法律依据**：《个保法》第21-23条（受托处理）、《数据安全法》第27-30条、《网络安全法》第21条。

**高频争议防范**：SLA口径争议→明确"可用性"定义和监测方式；数据泄露→明确"安全事件"定义和责任归属；续费锁定→退出条款在签约时即约定。

#### 【技术开发合同】

| 条款 | 起草要点 | 行业基准参数 |
|-----|---------|------------|
| 知识产权归属 | 严格区分"背景IP"与"前景IP"，委托开发须明确约定归属 | 法律默认专利申请权归开发方（第859条） |
| 源代码交付 | 完整源代码+编译环境+部署文档+接口文档+SBOM | 单测覆盖率≥60%，无Critical级漏洞 |
| 阶段验收 | 里程碑式验收：需求确认20-30% → 编码30-40% → 测试20-30% → 上线10-20% | 每阶段设明确交付物和验收期限 |
| 开源合规 | 保证不含GPL/AGPL/SSPL等强传染性组件，提供SBOM清单 | 高风险(GPL)=禁用，中风险(LGPL)=仅动态链接 |
| 需求变更 | 建立CCB机制：书面变更→影响评估→双方确认→调整工期费用 | 变更致费用增加>10%须另行签署补充协议 |

**关键法条**：委托开发IP第859条、合作开发IP第860-861条、技术合同无效第850条。

**高频争议防范**：IP未约定→法律默认归开发方，**务必明确约定**；需求变更→建立CCB机制；开源污染→交付前开源扫描并提供SBOM。

| 条款 | 起草要点 | 行业基准参数 |
|-----|---------|------------|
| 押金/保证金 | 明确金额、扣减情形、退还期限 | 商业2-3月租金，退还15-30个工作日 |
| 租金调整 | 固定涨幅/CPI联动/年度议价，涨幅上限 | 写字楼3%-5%/年，商铺Max(CPI,3%)/年且≤8% |
| 装修条款 | 免租装修期、折旧年限、各退出情形补偿规则 | 免租1-3月，摊销5-10年（直线法） |
| 提前退租 | 违约金标准、通知期、免租期租金补交 | 违约金2-6月租金，通知期3-6月 |
| 物业交还 | 交还标准（恢复原状vs现状）、设施清单、水电结清 | 逾期腾退按日租金1.5-2倍 |

**高频争议防范**：装修补偿→写明折旧年限和残值计算公式；提前退租→同时约定"违约金不足弥补损失的赔偿差额"；物业交还→签约时做现状确认书（附照片/视频）。

#### 【股权转让/投资合同】

| 条款 | 起草要点 | 行业基准参数 |
|-----|---------|------------|
| 对赌条款 | 回购型+现金补偿型，明确业绩计算规则和行权期间 | 回购年化8%-15%，不超LPR四倍；业绩对赌3-5年 |
| 陈述与保证 | 按类型设存续期：基础(无限期)、商业(12-24月)、税务(3-5年) | 起赔门槛+累计门槛(对价0.5%-1%)+Cap(对价15%-30%) |
| 交割条件 | 政府审批、工商变更、无MAC、陈述保证持续真实 | 交割期30-180天 |
| 竞业禁止 | 范围明确界定、期限2-5年、非招揽12-24月 | 须有合理对价 |

**对赌条款司法态度**：与股东对赌→有效可执行；与目标公司对赌→有效但须完成减资程序（九民纪要+最新司法解释）。建议同时约定股东回购+公司补偿。

**关键法条**：股权转让《公司法》第71条、对赌效力九民纪要第5条、违约责任第577条。

#### 【劳务外包/服务外包合同】

| 条款 | 起草要点 | 行业基准参数 |
|-----|---------|------------|
| 劳动关系归属 | 外包方为用人单位，独立行使劳动管理权，按成果验收 | "假外包真派遣"罚款每人5000-10000元 |
| 社保与工伤 | 外包方为缴费义务人，按月提供社保凭证，购买雇主责任险 | 雇主责任险≥100万元/人 |
| KPI联动 | 服务费=基础费(70-80%)+绩效(20-30%)，月度/季度考核 | 到岗率≥95%、流失率≤15%/年 |
| 人员替换 | 不胜任替换期限、过渡交接期 | 替换期5-15日，交接期7-15天 |

**"假外包真派遣"红线**：发包方直接管理外包人员、按人头/工时计费、外包人员受发包方规章制度约束。

### 第三步：逐条起草合同条款

基于内置知识引用核心法条，标注"[建议核实法条原文]"。引用格式：《民法典》第XXX条第X款。

**起草五大铁律（每个条款必须遵守）：**

**铁律一·量化原则**——消灭模糊表述。"及时付款"→"收到合规发票后15个工作日内付款"；"合理费用"→"不超过合同总额5%的费用"。

**铁律二·防弹设计**——每个义务对应明确违约后果；争议解决具体到仲裁机构全称或管辖法院；仲裁与法院二选一绝不并列。

**铁律三·举证预置**——交付确认约定签收单效力；通知到达约定送达方式（挂号信签收日/快递签收日/邮件发送次日视为送达）；损失证明约定特定凭证效力。

**铁律四·解释锁定**——关键术语唯一定义；完整协议条款排除口头承诺；鉴于条款锁定合同目的；效力优先顺序：补充协议 > 合同正文 > 附件。

**铁律五·合规底线**——格式条款提示义务（加粗/下划线，第496条）；违约金合理性（超过实际损失30%可能被调减）；定金不超过标的额20%（第586条）；约定仲裁不能同时约定法院管辖。

### 第四步：财税条款一体化设计

| 要素 | 标准设计 |
|-----|---------|
| 发票类型 | "乙方应向甲方开具税率为X%的增值税专用发票" |
| 开票时间 | "乙方应在付款日前X个工作日将合规发票送达甲方" |
| 先票后款 | "甲方收到发票并验证无误后方履行付款义务" |
| 税率变更 | 三选一：含税总价不变 / 不含税价不变随税调整 / 超X%协商 |
| 发票合规承诺 | 违规导致买方无法抵扣的赔偿全部税损+滞纳金+罚款 |

**含税vs不含税**：未明确约定的法院一般推定为含税价。建议同时列明含税总价、不含税价和税额。格式："合同总价人民币壹拾壹万叁仟元整（\113,000.00），其中不含税价壹拾万元整（\100,000.00），增值税税额壹万叁仟元整（\13,000.00），税率13%"。

### 第五步：填充待确认信息

未确认的商业参数使用行业惯例基准值填充，并标注为【待确认：XXX】高亮。示例：
- 【待确认：付款比例建议30%(预付)+65%(验收)+5%(质保金)】
- 【待确认：质保期建议12个月，自验收合格之日起算】
- 【待确认：逾期交付违约金建议每日万分之三，超30日可解除】
- 【待确认：知识产权归属建议归委托方所有（未约定则法律默认归开发方）】

### 第六步：格式化输出Word文档

使用 Node.js `docx` 库生成专业排版的 .docx 文件。详细排版参数见 [references/word-format-spec.md](references/word-format-spec.md)。

**核心排版要求速览**：
- 纸张A4，左边距30mm（含装订线），其余25mm
- 合同大标题：二号黑体加粗居中；条标题：四号黑体加粗；正文：四号仿宋，首行缩进2字符
- 签章页必须另起一页，双栏布局，预留盖章位置40mm x 40mm

## 违约金设计专项指引

**设计原则：锚定实际损失 + 阶梯递进 + 赔偿兜底**

**标准违约金模板**：
- 迟延1-15日：每日万分之三
- 迟延16-30日：每日万分之五
- 迟延超30日：守约方可解除合同并要求支付合同总价10%的违约金
- 根本违约：合同总价20%的违约金
- 兜底条款："上述违约金不足以弥补守约方实际损失的，违约方还应赔偿超出部分。"

**违约金比例安全区间（不易被法院调减）**：

| 类型 | 安全区间 | 风险阈值 | 法律依据 |
|-----|---------|---------|---------|
| 逾期违约金（日） | 万分之三至万分之五 | 超万分之五有调减风险 | 第585条+通则解释第64条 |
| 根本违约金 | 合同总价10%-20% | 超过实际损失30%可能被调减 | 通则解释第64条 |
| 定金 | 不超过标的额20% | 超过20%部分不生效 | 第586条硬性上限 |

**防法院调减技巧**——加入损失预估声明："双方确认，如一方违约，预计可能造成的损失包括直接损失、间接损失和向第三方承担的违约责任。基于上述合理预估，双方认为本合同约定的违约金比例是合理的。"

## 质量标准

- 所有关键术语在"定义与解释"条款中有精确定义，杜绝模糊表述
- 法条引用格式统一：《民法典》第XXX条第X款
- 金额同时使用大小写：人民币壹拾万元整（\100,000.00）
- 各条款逻辑自洽（约定仲裁不能同时约定法院管辖）
- 违约金设置在司法安全区间内
- 格式条款须以加粗/下划线等方式履行第496条提示说明义务
- 所有待确认事项以行业惯例基准值填充，并用【待确认：XXX】高亮标注
- 起草说明完整覆盖设计考量、风险提示和法律依据索引
- 每个条款都应能回答"为什么这样设计"——体现专业法律知识
