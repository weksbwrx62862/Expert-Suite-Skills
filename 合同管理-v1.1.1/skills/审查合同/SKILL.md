# /审查合同 -- 合同风险审查

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

对合同进行逐条风险审查，标记偏差，生成修改建议和谈判策略。

**重要提示**：本技能辅助法律工作流程，不构成法律意见。所有分析结果应经执业律师审核后方可作为决策依据。

**独立能力（无需连接器）**

- 12类条款逐条审查
- 红黄绿三级风险标记
- 具体替代条款措辞（非方向性建议）
- 三梯队谈判策略排序
- 缺失条款检测

**增强能力（连接器加持）**

- ~~Notion -> 将审查报告发布到 Notion，便于团队审阅

## 连接器（可选增强）

| 连接器 | 增强能力 |
|-----------|---------|
| **~~Notion** | 将审查报告发布到 Notion，便于团队批注讨论 |

> 没有连接器也完全可以使用——基于内置知识完成审查，法条标注"[建议核实原文]"。

## 调用方式

```
/审查合同 <合同文件>
```

审查此合同：@$1

接受文件上传（PDF、DOCX、图片扫描件）或直接粘贴合同文本。

## 输入要求

1. **我方立场**：甲方/乙方、采购方/供应商、委托方/受托方、许可方/被许可方
2. **合同类型**：采购合同/SaaS服务/技术开发/租赁/股权转让/劳务外包/代理经销/其他
3. **关注重点**（可选）：如"付款条件很关键"、"知识产权是核心问题"
4. **交易背景**（可选）：合同金额量级、战略重要性、是否长期合作

用户只提供部分信息也可继续，标注假设条件。

## 执行流程

### 第一步：识别合同基本信息

提取：合同类型、签约双方（含统一社会信用代码如有）、签约日期、合同期限、合同标的、合同金额。

### 第二步：签约方背景核查

提示用户自行核实对方主体资格，标注"[建议通过企查查/天眼查核实签约方主体状态]"。

如用户提供了对方工商信息：
- 确认主体真实存续
- 如对方存在重大风险信号（已注销、被列为失信被执行人、大量诉讼），在报告开头予以警示

### 第三步：加载合同类型专属审查重点

不同合同类型的审查侧重差异显著，按类型加载专属审查焦点：

| 合同类型 | 必审重点（Top 3） | 高频风险条款 |
|---------|-----------------|------------|
| **采购合同** | 验收标准、质保期、付款条件 | 默示验收陷阱、质量举证责任倒置、逾期交付阶梯违约金 |
| **SaaS服务** | SLA服务等级、数据安全归属、退出迁移 | 自动续费锁定、数据删除义务、赔偿以代金券抵扣 |
| **技术开发** | 知识产权归属、源代码交付、验收程序 | IP未约定默认归开发方、开源组件污染、需求变更无CCB |
| **租赁合同** | 押金退还、租金调整机制、提前退租 | 装修补偿模糊、转租限制、物业交还标准不明 |
| **股权转让** | 对赌条款、陈述保证、交割条件 | 对赌与目标公司直接对赌执行风险、MAC条款模糊 |
| **劳务外包** | 劳动关系归属、社保工伤、假外包真派遣 | 发包方直接管理、按人头计费、外包人员受发包方制度约束 |

### 第四步：逐条审查

基于内置知识引用核心法条，标注"[建议核实原文]"。

通读全文后，按以下审查表逐条分析。条款之间存在联动关系（如违约金与损害赔偿互相影响），需整体评估风险分配的合理性。

| 条款类别 | 审查要点 |
|---------|---------|
| **合同主体** | 签约主体资格、是否需要授权委托书、关联关系披露 |
| **标的与范围** | 标的物/服务内容是否明确、验收标准、交付条件 |
| **价款与支付** | 付款条件、付款节点、发票类型、含税/不含税、逾期付款违约金 |
| **违约责任** | 违约金上限（参照第585条）、损害赔偿范围、免责条款 |
| **知识产权** | 背景IP归属、新产生IP归属（第847/848条）、职务作品、开源组件 |
| **保密条款** | 保密范围、期限、例外情形、违约救济 |
| **不可抗力** | 定义范围、通知义务、后果处理 |
| **争议解决** | 约定管辖vs法定管辖、仲裁条款有效性、适用法律 |
| **合同期限** | 期限起算、续约条件、提前终止权、到期后义务存续 |
| **担保与保证** | 保证金/质保金、担保方式、保证人资格 |
| **陈述与保证** | 范围、存续期间、违反后果 |
| **转让与变更** | 转让限制、变更程序、通知义务 |

**缺失条款检测**——除审查现有条款外，还须检查应有但缺失的关键条款：

| 应有条款 | 缺失后果 | 适用合同类型 |
|---------|---------|------------|
| 通知送达条款 | 无法证明通知已送达，影响解除权行使 | 所有类型 |
| 完整协议条款 | 口头承诺可能被主张为合同内容 | 所有类型 |
| 不可抗力条款 | 适用法定规则，可能不符合商业预期 | 所有类型 |
| 数据安全条款 | 数据泄露无明确责任分配 | SaaS/技术开发 |
| 退出迁移条款 | 合同终止后数据迁移无保障 | SaaS |
| 源代码交付条款 | 无法获取源代码进行自主维护 | 技术开发 |
| 物业交还标准 | 退租时交还状态争议 | 租赁 |

### 第五步：标记风险等级

**通过（绿色）**——条款合理，无需修改。

**需改（黄色）**——条款偏离我方理想立场但可谈判。
必须输出：原文引用 + 替代条款措辞 + 修改理由 + 退让底线。

**需谈（红色）**——重大风险或严重偏离法律规定。
必须输出：具体风险描述 + 法律依据 + 可能的经济/法律后果 + 谈判策略或替代方案。

**违约金合理性量化校验**（适用于所有包含违约金的条款）：

| 校验项 | 安全区间 | 风险阈值 | 法律依据 |
|-------|---------|---------|---------|
| 逾期违约金（日） | 万分之三至万分之五 | 超过万分之五有被调减风险 | 第585条+通则解释第64条 |
| 根本违约金 | 合同总价10%-20% | 超过30%可能被调减 | 超过实际损失30%的部分 |
| 定金 | 不超过主合同标的额20% | 超过20%部分不生效 | 第586条硬性上限 |
| 对方违约金vs我方违约金 | 比例对等 | 差距>2倍视为不公平 | 公平原则 |

### 第六步：生成谈判策略

按优先级组织修改建议：

- **第一梯队（必须争取）**：涉及合同效力、重大经济风险、法律强制性规定的条款
- **第二梯队（重点争取）**：影响权利义务平衡但有谈判空间的条款
- **第三梯队（可以让步）**：改善我方地位但非关键的条款，可作为谈判筹码

### 第七步：输出报告

**如已连接 ~~Notion：**
- 将审查报告发布到 Notion，便于团队批注讨论

**如未连接 ~~Notion：**
- 以对话形式输出报告

## 质量标准

- 每项审查意见必须引用具体法条或司法解释条文
- 修改建议必须提供具体替代条款措辞，非笼统方向性建议
- 风险等级判定必须有明确理由
- 违约金审查必须按量化校验规则检查合理性
- 不得引用外国法律作为主要依据
- 仲裁条款必须检查有效性三要素（仲裁意思表示、仲裁事项、仲裁机构）
- 缺失条款检测必须覆盖，不能只审查现有条款
- 双方违约责任对等性必须检查

## 关联技能

- **合同对比** -- 如有对方修改的新版本，可对比两版差异
- **NDA快筛** -- 如合同中含有独立NDA或保密条款需要快速评估
- **法条速查** -- 深入了解特定条款涉及的法条细节
- **合同台账提醒** -- 审查通过后录入台账管理到期提醒
