# /合同对比 -- 合同版本红线对比

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

对两份合同进行条款级精细语义对比，生成红线差异报告，标注每处变更的实质影响。

**重要提示**：本技能辅助法律工作流程，不构成法律意见。

**独立能力（无需连接器）**

- 语义级对比（非简单文字差异）
- 三级影响评估（低/中/高）
- 联动分析（组合风险检测）
- 变更态势分析和谈判建议
- 对方修改动机推断

**增强能力（连接器加持）**

- ~~Notion -> 将对比报告发布到 Notion，便于团队批注讨论

## 调用方式

```
/合同对比 <两份合同文件>
```

对比这两份合同：@$1

接受文件上传（PDF、DOCX）或直接粘贴合同文本。需提供两份合同。

## 输入要求

用户需提供两份合同文件：
- **基准版**：原始版本 / 我方版本 / 上一轮版本
- **对比版**：对方修订版 / 最新版本 / 当前谈判版

如用户只上传一份文件，询问另一份。确认：哪份是基准版，哪份是对比版？

## 执行流程

### 第一步：文本提取与结构化

读取两份合同全文。识别章节结构，按功能类别进行结构化拆分：

| 类别 | 包含条款 |
|------|---------|
| 主体条款 | 合同当事人、资格条件、授权代表 |
| 标的条款 | 标的物/服务范围、技术规格、验收标准 |
| 商务条款 | 价款、付款、发票、税费 |
| 权利保护条款 | 知识产权、保密、竞业限制 |
| 风险分配条款 | 违约责任、赔偿上限、不可抗力、保险 |
| 期限条款 | 合同期限、续约、提前终止 |
| 争议解决条款 | 管辖、仲裁、适用法律 |

建立两版之间的条款对照关系，处理常见情况：条款编号变动（按内容匹配）、新增/删除整个条款、条款位置调整、附件变更、定义条款变更（影响全文）。

### 第二步：逐条语义级对比

对每个条款进行语义级对比，核心判断——变更是否改变了权利义务关系：

**文字修改**（低影响）：措辞调整但实质含义不变。
注意伪文字修改：看似措辞调整实则含义不同，如"可以"改为"应当"、"或"改为"且"、"合理"改为"必要"。

**实质修改**（中/高影响）：从五个维度评估实质影响：

| 评估维度 | 关注重点 | 高影响判定 |
|---------|---------|-----------|
| 权利义务转移 | 单方权利变双方义务、义务加重 | 核心权利被削弱或核心义务被加重 |
| 风险分配变化 | 违约金、赔偿上限、免责条款 | 风险敞口显著扩大 |
| 经济条件变化 | 价款、付款节点、付款条件 | 金额变动>10%或付款条件实质收紧 |
| 时间节点变化 | 履行期限、验收期限、质保期 | 时间变动>30%或关键节点前移 |
| 违约后果变化 | 违约金比例、解除权触发 | 新增单方解除权或违约后果加重 |

**新增条款**：重点评估是否对我方不利，特别关注排他性、竞业限制、审计权、单方解除权。

**删除条款**：重点评估我方是否失去重要保护，特别关注保护性条款被删除。

**数字变更**：必须同时列出原值、新值、变动绝对值和变动百分比。

### 第三步：影响评估

基于内置知识评估法律影响，涉及法条标注"[建议核实原文]"。

影响等级判定标准：
- **低**：表述优化，不改变权利义务，汇总即可
- **中**：条件范围变化，需评估经济影响，逐条列出
- **高**：核心条款重大变化，必须逐条分析法律后果和经济后果

### 第四步：联动分析

必须检查各处变更之间的联动关系——单看合理的修改组合可能构成重大风险：

| 联动组合 | 组合风险 | 检查要点 |
|---------|---------|---------|
| 违约金 + 赔偿上限 | 风险敞口变化 | 降低违约金同时降低赔偿上限=实质免责 |
| 知识产权归属 + 保密条款 | 技术成果控制力 | IP归对方+宽松保密=技术流失风险 |
| 付款条件 + 验收标准 | 付款实质收紧 | 验收标准提高+付款挂钩验收=延迟付款 |
| 解除条件 + 合同期限 | 合同稳定性 | 新增单方解除权+缩短通知期=随时甩单 |
| 竞业限制 + 排他性 | 业务限制范围 | 竞业范围扩大+排他期延长=过度限制 |
| 担保/保证金 + 违约条件 | 资金占用风险 | 保证金增加+扣减条件放宽=变相涨价 |
| 不可抗力 + 免责条款 | 风险转嫁 | 不可抗力范围扩大+新增免责事由=对方几乎不承担风险 |

### 第五步：变更态势与动机分析

分析对方修改的整体倾向：

| 态势类型 | 典型表现 | 谈判策略 |
|---------|---------|---------|
| 强势压制 | 大量加重我方义务、削弱我方权利、新增单方权利 | 集中反击核心条款，次要条款适度让步换取核心利益 |
| 风险转嫁 | 扩大免责范围、降低赔偿上限、增加不可抗力事由 | 要求对等安排，逐条还原风险平衡 |
| 技术收紧 | 抬高验收标准、缩短履行期限、增加报告义务 | 评估可行性，对不合理要求提供行业基准反驳 |
| 整体均衡 | 双方互有让步，无明显偏向 | 聚焦剩余分歧点快速推进 |

### 第六步：生成对比报告

**如已连接 ~~Notion：**
- 将对比报告发布到 Notion，便于团队批注讨论

**如未连接 ~~Notion：**

按影响等级从高到低排列。

## 质量标准

- 不得遗漏任何实质性变更——漏报比误报的后果更严重
- 严格区分语义级差异和文字级差异，伪文字修改必须识别
- 高影响变更必须附具体理由和法条引用
- 新增和删除条款必须独立标注，不混入修改列表
- 数字变更必须同时列出原值、新值和变动百分比
- 建议必须明确（接受/拒绝/协商），附具体理由
- 联动分析必须覆盖上述7组高频联动组合
- 变更态势分析须准确反映对方修改的整体倾向

## 关联技能

- **审查合同** -- 对对比版做完整的风险审查
- **法条速查** -- 深入了解高影响变更涉及的法条
- **起草合同** -- 起草我方的反提案版本
