# Skill 迭代指南

专家套件不是一次性的。封装完成后，需要根据实际使用反馈持续迭代优化。

## 迭代节奏

### 第一阶段：首次封装（Week 1）

**目标**：覆盖岗位最核心的3-5个场景，形成最小可用套件。

**做法**：
- 选择使用频率最高的工作场景
- 每个场景封装为一个独立Skill
- 确保无连接器时可独立运行
- 完成plugin.json、README.md等套件级文件

**质量标准**：
- [ ] 每个Skill有明确的触发词（至少3个同义表达）
- [ ] 每个Skill有完整的输出模板
- [ ] 套件级文件完整（plugin.json、README.md、CONNECTORS.md）
- [ ] 所有Skill在无连接器时可独立运行

### 第二阶段：使用验证（Week 2-3）

**目标**：实际使用1-2周，记录问题和改进点。

**做法**：
- 每天使用套件完成实际工作
- 记录以下问题：
  - 触发词是否被正确识别？
  - 输出是否符合预期？
  - 哪些步骤需要人工干预？
  - 输出质量是否达到业务标准？
- 收集同事反馈

**记录模板**：

```
日期：2026-04-28
Skill：审查合同
输入：采购合同，我方为甲方
问题：
1. 触发词"看看合同"未被识别，需要说"审查合同"才触发
2. 输出中违约金审查缺少量化校验
3. 谈判策略过于笼统，没有针对采购合同的具体建议
改进：
1. 在description中补充"看看合同"作为触发词
2. 在质量标准中增加违约金量化校验规则
3. 在谈判策略中增加采购合同专属策略
```

### 第三阶段：补充完善（Week 4-5）

**目标**：根据验证阶段的问题，优化现有Skill并补充新Skill。

**优化方向**：

| 问题类型 | 优化方法 | 示例 |
|---------|---------|------|
| 触发词遗漏 | 补充同义表达 | "看看合同" → 加入description |
| 输出不完整 | 补充输出模板 | 增加违约金量化校验表格 |
| 流程有漏洞 | 补充执行步骤 | 增加缺失条款检测环节 |
| 质量不达标 | 补充质量标准 | 增加"不得笼统表述"规则 |
| 场景覆盖不足 | 新增Skill | 增加"合同对比"Skill |

**新增Skill原则**：
- 新增Skill必须解决一个独立的工作场景
- 新增Skill与现有Skill有明确的关联关系
- 新增Skill遵循相同的结构和质量标准

### 第四阶段：版本更新（Week 6+）

**目标**：发布新版本，记录变更内容。

**版本号规则**（semver）：

| 变更类型 | 版本号变化 | 示例 |
|---------|-----------|------|
| Bug修复、文案优化 | 修订号+1 | 1.0.0 → 1.0.1 |
| 新增Skill、功能增强 | 次版本号+1 | 1.0.1 → 1.1.0 |
| 重大架构调整 | 主版本号+1 | 1.1.0 → 2.0.0 |

**变更记录**：

在README.md或单独CHANGELOG中记录：

```
## v1.1.0 (2026-05-15)

### 新增
- 新增"合同对比"Skill，支持两版合同差异比对

### 优化
- "审查合同"Skill：补充"看看合同"等口语化触发词
- "审查合同"Skill：增加违约金量化校验规则
- "起草合同"Skill：优化Word排版参数

### 修复
- "法条速查"Skill：修复部分法条引用格式错误
```

## 持续优化信号

以下信号出现时，说明需要迭代：

| 信号 | 说明 | 行动 |
|------|------|------|
| 用户多次用不同说法尝试触发 | 触发词覆盖不足 | 补充同义表达 |
| 用户每次都要补充相同信息 | 输入要求不完整 | 补充必填项或默认值 |
| 用户反馈"输出太笼统" | 输出模板不具体 | 补充占位符和示例 |
| 用户手动修改输出后使用 | 输出不符合业务标准 | 优化质量标准 |
| 用户跳过某些步骤 | 流程设计不合理 | 简化或重组流程 |
| 新增业务场景 | 套件覆盖不足 | 新增Skill |

## 迭代检查清单

每次迭代前检查：

- [ ] 已收集至少3条使用反馈
- [ ] 已记录具体问题和改进点
- [ ] 改进点已按优先级排序
- [ ] 版本号已按semver规则更新
- [ ] 变更内容已记录

## 版本兼容性

- **修订号更新**：用户无感知，直接替换
- **次版本号更新**：新增Skill，不影响现有Skill使用
- **主版本号更新**：可能有破坏性变更，需要通知用户
