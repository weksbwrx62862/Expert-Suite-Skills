# 示例：HR招聘套件（从0到完整套件）

本示例展示如何从零开始创建一个完整的专家套件，遵循速查式指导的规范。

## Step 1: 岗位分析

| 维度 | 内容 |
|------|------|
| **岗位** | HR招聘专员 |
| **核心问题** | 简历筛选效率低、面试评估不标准、Offer谈判缺数据、入职跟进易遗漏 |
| **高频场景** | 每天筛20-50份简历、每周5-10场面试、每月2-5次Offer谈判 |
| **工具** | Notion（候选人库）、邮件（Offer发送） |
| **输出格式** | 匹配度评分报告、评估表、薪资方案、入职清单 |

## Step 2: Skill清单

| Skill | 功能 | 触发词示例 |
|-------|------|-----------|
| 简历筛选 | 提取信息，匹配JD | "筛简历"、"看简历"、"这个简历怎么样" |
| 面试评估 | 按维度打分，生成报告 | "面试评估"、"写面评"、"评估候选人" |
| Offer谈判 | 薪资对标，生成方案 | "谈Offer"、"定薪资"、"Offer方案" |
| 入职跟进 | 检查清单，生成计划 | "入职准备"、"新人跟进"、"入职清单" |

**工作流**：
```
简历筛选 ──→ 面试评估 ──→ Offer谈判 ──→ 入职跟进
```

```markdown
# /简历筛选 -- 简历智能筛选

> 如遇到不熟悉的占位符，参见 [CONNECTORS.md](../../CONNECTORS.md)。

**独立能力（无需连接器）**

- 关键信息提取（教育、工作、项目、技能）
- JD匹配度评分（0-100分）
- 亮点和风险点标注
- 面试问题建议

**增强能力（连接器加持）**

- ~~Notion → 将筛选结果写入候选人数据库

## 连接器（可选增强）

| 连接器 | 增强能力 |
|--------|---------|
| **~~Notion** | 将筛选结果写入Notion候选人库 |

> 没有连接器也完全可以使用——基于上传的简历和JD进行分析。

## 调用方式

```
/简历筛选 <简历文件+JD>
```

筛选此简历：@$1

接受PDF、DOCX、图片格式的简历文件，JD可以直接粘贴文本。

## 输入要求

1. **简历文件**（必填）：候选人简历
2. **JD内容**（必填）：岗位描述，含职责和要求
3. **关注重点**（可选）：如"重点关注Python经验"

## 执行流程

### 第一步：信息提取

从简历中提取：
- **基本信息**：姓名、联系方式、工作年限
- **教育背景**：学历、学校、专业
- **工作经历**：公司、职位、时间、核心职责
- **项目经验**：项目名、角色、技术栈、成果
- **技能清单**：专业技能、语言能力、证书

### 第二步：JD解析

从JD中提取：
- **硬性要求**：学历、工作年限、必备技能
- **软性要求**：沟通能力、团队协作
- **加分项**：额外技能、行业经验
- **排除项**：明确要求不满足即排除

### 第三步：匹配度评分

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 硬性要求 | 40% | 每项不满足扣10-20分 |
| 技能匹配 | 30% | 必备技能覆盖度 |
| 经验匹配 | 20% | 行业/岗位经验相关性 |
| 加分项 | 10% | 额外匹配度 |

**评分等级**：
- 90-100分：强烈推荐
- 75-89分：推荐面试
- 60-74分：可考虑
- 60分以下：不匹配

### 第四步：亮点与风险

**亮点**：超出JD要求的经验、知名公司背景、量化成果突出

**风险**：工作频繁变动、经验与JD有差距、简历中的模糊表述

### 第五步：面试问题建议

生成3-5个针对性问题：
- 验证风险点的问题
- 深入了解亮点的问题
- 考察文化匹配度的问题

## 输出格式

```
## 简历筛选报告

**候选人**：[姓名]
**岗位**：[岗位名称]
**匹配度评分**：[X]/100 [等级]

### 基本信息
| 项目 | 内容 |
|------|------|
| 工作年限 | [X年] |
| 学历 | [学历/学校/专业] |
| 当前公司 | [公司/职位] |

### 匹配度分析
| 维度 | 得分 | 说明 |
|------|------|------|
| 硬性要求 | [X]/40 | [说明] |
| 技能匹配 | [X]/30 | [说明] |
| 经验匹配 | [X]/20 | [说明] |
| 加分项 | [X]/10 | [说明] |

### 亮点
1. [亮点1]
2. [亮点2]

### 风险
1. [风险1] — 建议面试中验证：[问题]
2. [风险2] — 建议面试中验证：[问题]

### 面试问题建议
1. [问题1] — 目的：[考察点]
2. [问题2] — 目的：[考察点]

### 建议
[推荐/不推荐/待定] — [理由]
```

## 质量标准

- 评分必须有明确的维度分解，不能只有一个总分
- 风险点必须附带验证问题，不能只列风险
- 亮点必须是简历中客观存在的信息，不能臆测
- 不匹配的情况必须明确说明哪项硬性要求不满足
- 不得因性别、年龄、籍贯等非相关因素进行评价

## 关联Skill

- **面试评估** — 筛选通过后，用 `/面试评估` 进行结构化面试
```
