# 专家套件质量检查清单

创建或迭代专家套件时，逐项检查以下项目。每个检查项都附带验证方法。

## Skill 级检查

### Frontmatter

| 检查项 | 验证方法 |
|--------|---------|
| [ ] `name` 字段已填写，与文件夹名称一致 | 打开文件夹，检查SKILL.md中的name是否与文件夹名相同 |
| [ ] `description` 字段已填写，超过30字 | 统计description字符数，检查是否>30 |
| [ ] `description` 包含明确的触发词（至少3个） | 列出description中所有引号内的词组，检查是否>=3个 |
| [ ] `argument-hint` 已填写，帮助用户理解输入格式 | 检查是否包含输入示例（如"上传合同文件"） |
| [ ] `name_en` 已填写（如套件面向多语言用户） | 检查是否为kebab-case格式 |
| [ ] `description_en` 已填写，包含英文触发词 | 列出英文触发词，检查是否>=3个 |

### 能力说明

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 明确列出"独立能力"（无连接器时可用） | 检查是否有"独立能力"section，且内容>=3项 |
| [ ] 明确列出"增强能力"（连接器加持后可用） | 检查是否有"增强能力"section |
| [ ] 独立能力至少3项 | 数独立能力列表中的条目数 |
| [ ] 没有连接器也完全可以使用——已明确说明 | 搜索"> 没有连接器也完全可以使用"或类似说明 |

### 连接器

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 连接器表格已填写（即使无连接器也要说明） | 检查是否有"## 连接器"section和表格 |
| [ ] 占位符使用 `~~ConnectorName` 格式 | 搜索"~~"，检查格式是否统一 |
| [ ] 已说明无连接器时的工作方式 | 检查是否有说明无连接器时的替代方案 |
| [ ] 连接器仅作增强，不是必需 | 检查核心功能是否都能在"独立能力"中完成 |

### 调用方式

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 提供了 `/命令 <参数>` 格式 | 检查是否有代码块包含`/Skill名称`格式 |
| [ ] 说明了 `@$1` 的用途 | 检查是否有"@$1"并说明其含义 |
| [ ] 说明了接受的输入格式（文件类型、文本等） | 检查是否列出接受的文件类型（PDF/DOCX/图片） |

### 输入要求

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 列出了必填信息 | 检查是否有加粗的"必填"或"必填项"标记 |
| [ ] 列出了选填信息 | 检查是否有"选填"或"可选"标记 |
| [ ] 说明了信息缺失时的处理方式 | 检查是否有"用户只提供部分信息也可继续"等说明 |

### 执行流程

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 按步骤编号，每步有明确标题 | 检查是否有"### 第一步"、"### 第二步"等 |
| [ ] 每步有具体的处理逻辑 | 随机选一步，检查是否有具体的子步骤或说明 |
| [ ] 复杂判断用表格呈现 | 检查是否有条件判断表格（条件/处理方式） |
| [ ] 包含分支处理（如适用） | 如果Skill有多个场景，检查是否有分支 |
| [ ] 步骤之间逻辑连贯 | 读完整流程，检查步骤之间是否有逻辑跳跃 |

### 输出格式

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 提供了完整的输出模板 | 检查是否有代码块包含完整的输出结构 |
| [ ] 模板中包含占位符说明 | 检查是否有`[值]`、`[X]`等占位符 |
| [ ] 使用了表格、代码块、列表等多种格式 | 检查输出模板中是否包含至少2种格式 |
| [ ] 输出模板可直接使用，不是半成品 | 检查是否有"..."或"等等"等不完整表述 |

### 质量标准

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 列出了可验证的质量要求 | 检查每条标准是否可以用yes/no判断 |
| [ ] 包含常见错误和正确做法对照（如适用） | 检查是否有"反模式"或"常见错误"表格 |
| [ ] 标注了红线规则（绝对不可违反） | 检查是否有"不得"、"必须"等绝对性表述 |
| [ ] 质量标准不少于3条 | 数质量标准列表中的条目数 |

**质量标准可验证性测试**：

| 差示例 | 好示例 | 验证方法 |
|--------|--------|---------|
| "输出要专业" | "法条引用必须精确到条款项" | 检查输出中每条法条是否精确到条/款/项 |
| "格式要规范" | "金额必须同时使用大小写" | 检查金额是否同时有大写和小写 |
| "分析要深入" | "ROE分析须用杜邦分解" | 检查ROE分析是否分解为三个因子 |

### 关联Skill

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 列出了套件内相关的Skill（如适用） | 检查是否有"## 关联Skill"section |
| [ ] 说明了使用场景和衔接方式 | 检查是否说明了何时调用关联Skill |

## 套件级检查

### plugin.json

| 检查项 | 验证方法 |
|--------|---------|
| [ ] `name` 符合 kebab-case | 检查是否只包含小写字母、数字和短横线 |
| [ ] `displayName` 已填写，简洁明了 | 检查字符数是否<10 |
| [ ] `version` 符合 semver | 检查格式是否为X.Y.Z |
| [ ] `description` 超过50字 | 统计字符数 |
| [ ] `author.name` 已填写 | 检查是否有author对象和name字段 |
| [ ] `keywords` 包含5-15个关键词 | 数keywords数组长度 |
| [ ] `skills` 数组与文件夹名称完全一致 | 逐项比对skills数组和skills/下的文件夹名 |
| [ ] `displayNameEn` 已填写 | 检查是否存在 |
| [ ] `descriptionEn` 已填写 | 检查是否存在 |
| [ ] JSON格式有效 | 使用JSON解析器验证 |

### .mcp.json

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 格式正确，包含 `mcpServers` 对象 | 检查顶层是否有mcpServers键 |
| [ ] 每个服务有 `type` 和 `url`（或 `command`/`args`） | 检查每个服务配置是否完整 |
| [ ] 服务标识与 SKILL.md 中的占位符一致 | 比对.mcp.json中的服务名和SKILL.md中的~~服务名 |

### CONNECTORS.md

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 说明了占位符的使用方式 | 检查是否有"~~"的说明 |
| [ ] 列出了本套件使用的连接器 | 检查是否有连接器表格 |
| [ ] 说明了无连接器时的工作方式 | 检查是否有"无连接器时"section |
| [ ] 说明了安装连接器后的增强效果 | 检查是否有增强效果说明 |

### README.md

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 标题为套件中文名 | 检查第一个#标题 |
| [ ] 描述一句话说明核心价值和覆盖场景 | 检查第一段是否<100字 |
| [ ] 包含免责声明 | 搜索"免责声明"或"不替代专业意见" |
| [ ] 包含能力层级图（ASCII图示） | 检查是否有ASCII框图 |
| [ ] 列出了适用角色（至少2个） | 数角色列表中的条目数 |
| [ ] 包含Skill工作流图 | 检查是否有Skill之间的箭头连接 |
| [ ] 快捷命令表格包含所有Skill | 数表格行数是否等于Skill数量 |
| [ ] 说明了MCP增强状态 | 检查是否有MCP或连接器说明 |

### 目录结构

| 检查项 | 验证方法 |
|--------|---------|
| [ ] `.qoder-plugin/plugin.json` 存在 | 检查文件是否存在 |
| [ ] `.mcp.json` 存在 | 检查文件是否存在 |
| [ ] `CONNECTORS.md` 存在 | 检查文件是否存在 |
| [ ] `README.md` 存在 | 检查文件是否存在 |
| [ ] `skills/` 文件夹存在 | 检查文件夹是否存在 |
| [ ] 每个Skill有独立的文件夹 | 检查skills/下的子文件夹数量是否等于skill数量 |
| [ ] 每个Skill文件夹下有 `SKILL.md` | 随机抽查几个Skill文件夹 |
| [ ] `skills` 数组中的名称与文件夹名称一致 | 逐项比对 |

## 内容质量检查

### 专业性

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 内容基于实际业务场景 | 检查是否有具体的业务术语和流程 |
| [ ] 使用了岗位专业术语 | 检查是否有行业专用词汇 |
| [ ] 流程和判断标准符合行业实践 | 检查判断标准是否与行业常识一致 |
| [ ] 输出格式符合行业规范 | 检查输出格式是否与行业标准一致 |

### 可用性

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 触发词覆盖了用户的常见表达方式 | 模拟3种不同说法，检查是否都能触发 |
| [ ] 输入要求清晰 | 检查用户是否知道该提供什么 |
| [ ] 输出模板完整 | 检查输出是否可以直接使用 |
| [ ] 质量检查清单可逐项验证 | 检查每条标准是否有明确的yes/no判断 |

### 一致性

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 套件内所有Skill的风格一致 | 对比2-3个Skill的结构是否相同 |
| [ ] 术语使用一致 | 搜索同一概念的不同说法 |
| [ ] 格式规范一致 | 检查表格、代码块等格式是否统一 |
| [ ] 连接器引用方式一致 | 检查~~ConnectorName格式是否统一 |

### 完整性

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 覆盖了岗位的核心工作场景 | 检查是否覆盖了该岗位80%的高频工作 |
| [ ] 每个Skill有完整的执行流程 | 检查是否有开始和结束 |
| [ ] 每个Skill有输出模板 | 检查是否有"## 输出格式"section |
| [ ] 每个Skill有质量标准 | 检查是否有"## 质量标准"section |

## 最终验证

### 无连接器测试

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 所有Skill在无连接器时可以独立运行 | 检查"独立能力"section是否包含核心功能 |
| [ ] 无连接器时的输出仍然完整可用 | 检查无连接器时的输出模板是否完整 |
| [ ] 无连接器时的质量不受显著影响 | 对比独立能力和增强能力，检查差距是否过大 |

### 示例验证

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 用具体案例测试每个Skill的执行流程 | 模拟一个真实输入，检查流程是否顺畅 |
| [ ] 检查输出是否符合模板 | 对比实际输出和输出模板 |
| [ ] 检查质量是否达到标准 | 对照质量标准逐项检查 |

### 关联验证

| 检查项 | 验证方法 |
|--------|---------|
| [ ] 测试Skill之间的串联使用 | 检查从SkillA到SkillB的衔接是否自然 |
| [ ] 检查工作流是否顺畅 | 按照README中的工作流图走一遍 |
| [ ] 检查衔接是否自然 | 检查关联Skill的说明是否清晰 |
