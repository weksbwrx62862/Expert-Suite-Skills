# README.md 标准模板

复制此模板创建套件的README.md。

```markdown
# 套件中文名

一句话描述套件的核心价值和覆盖场景。

> **免责声明：** 本插件辅助专业工作流程，不替代专业意见。所有输出应由具备相应资质的专业人士审核后方可用于决策。

## 能力层级

```
┌──────────────────────────────────────────────────────┐
│  ALWAYS（独立工作）                                    │
│  ✓ 能力1的简要描述                                     │
│  ✓ 能力2的简要描述                                     │
│  ✓ 能力3的简要描述                                     │
└──────────────────────────────────────────────────────┘
```

## 适用角色

- **角色1** — 核心使用场景描述
- **角色2** — 核心使用场景描述
- **角色3** — 核心使用场景描述

## Skill 工作流

```
SkillA ──→ SkillB ──→ SkillC（输出）
  ↑              ↑
SkillD ──→ SkillE
```

工作流说明：
- SkillA → SkillB：前置关系说明
- SkillB → SkillC：输出流转说明
- SkillD：并行输入说明

## 快捷命令

| 命令 | 说明 | 典型输入 |
|------|------|---------|
| `/Skill名称1` | 功能说明 | 输入示例 |
| `/Skill名称2` | 功能说明 | 输入示例 |
| `/Skill名称3` | 功能说明 | 输入示例 |

## MCP 增强

本插件独立工作，无需 MCP 连接器。（或：可选连接XXX增强YYY能力）
```

---

## 模板填写说明

### 标题和描述

- 标题用套件中文名
- 描述一句话说明核心价值和覆盖场景
- 必须包含免责声明

### 能力层级

用ASCII图示展示套件的核心能力：
- `ALWAYS（独立工作）`：无连接器时也能完成的能力
- 用 `✓` 列出3-5个核心能力

### 适用角色

列出2-4个主要使用角色：
- 角色名称用粗体
- 用 `—` 连接使用场景
- 场景要具体，不要泛泛而谈

### Skill 工作流

用ASCII图示展示Skill之间的流转关系：
- `──→` 表示前置依赖
- 并行流程用上下排列
- 在图示下方用文字说明关键流转节点

### 快捷命令

表格列出所有Skill命令：
- `命令`：斜杠命令格式
- `说明`：一句话功能描述
- `典型输入`：用户通常输入什么内容

### MCP 增强

说明连接器的状态：
- 如果不需要连接器：`本插件独立工作，无需 MCP 连接器。`
- 如果有可选连接器：`可选连接 Notion 增强知识库沉淀能力。`
