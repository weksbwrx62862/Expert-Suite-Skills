# /增值税管理 -- 增值税税负测算与申报

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

增值税管理工具，涵盖税负测算、进销项匹配检查、异常预警、申报底稿生成，以及出口退税、加计抵减、即征即退等特殊政策。

**重要提示**：本技能基于增值税法规和申报规则，税率和计算规则有明确法规依据，输出可验证。增值税优惠政策有时效性，须关注政策有效期。

## 能力层级

**独立能力（无需连接器）**：
- 销项税额计算和税率适用判定
- 进项税额核查和不可抵扣进项识别
- 税负率测算和行业预警
- 进销项匹配检查和发票风险识别
- 出口退税/加计抵减/即征即退适用性判断
- 申报底稿生成

**增强能力（连接器加持）**：
- 连接 ~~Notion → 将增值税分析报告和申报底稿写入 Notion 页面

## 连接器（可选增强）

| 连接器 | 增强能力 |
|--------|---------|
| **Notion** | 将增值税分析报告和申报底稿写入 Notion 页面，方便团队共享 |

> 没有连接器也完全可以使用——上传 Excel 即可完成分析。

## 调用方式

```
/增值税管理 <进销项数据或税务问题>
```

帮我看一下增值税情况：@$1

接受进销项明细数据（Excel）、发票数据或税务申报表。

## 输入要求

确认：
1. **纳税人类型**：一般纳税人 / 小规模纳税人
2. **所属行业**：影响税负率预警标准和特殊政策适用
3. **数据类型**：进销项明细 / 发票数据 / 申报表
4. **分析期间**：月度 / 季度 / 年度
5. **特殊业务**（如有）：出口业务 / 即征即退 / 加计抵减 / 差额征税

> **输入分支**：
> - 一般纳税人 + 完整进销项数据 → 完整五步分析
> - 小规模纳税人 → 简化流程（无进项抵扣），侧重销项税额和申报底稿
> - 有出口业务 → 增加出口退税计算模块
> - 软件/动漫企业 → 增加即征即退测算
> - 生活性/生产性服务业 → 增加加计抵减测算
> - 仅咨询税率适用问题 → 直接输出税率判定建议 + 依据

## 执行流程

### 第一步：数据获取

**如果连接了 ~~Notion：**
1. 从 Notion 数据库读取进销项台账数据
2. 分析结果可写入 Notion 页面

**如果未连接：**
1. 从用户上传的 Excel 中提取数据
2. 确认数据完整性

### 第二步：销项税额计算

**税率适用表**：

| 税率 | 适用范围 |
|------|---------|
| 13% | 销售货物 / 加工修理修配 / 有形动产租赁 |
| 9% | 交通运输 / 邮政 / 基础电信 / 建筑 / 不动产租赁 / 销售不动产 / 转让土地使用权 / 农产品 |
| 6% | 金融 / 现代服务 / 生活服务 / 增值电信 / 销售无形资产 |
| 0% | 出口货物 / 跨境应税服务 |
| 1% | 小规模纳税人减按1%（阶段性优惠，须确认有效期） |

**混合销售 vs 兼营决策框架**：

| 判断要素 | 混合销售 | 兼营 |
|---------|---------|------|
| 业务关系 | 一项销售行为同时涉及货物和服务 | 不同销售行为分别涉及不同税率项目 |
| 税率确定 | 按主业适用税率 | 分别核算分别适用税率；未分别核算的从高适用 |
| 典型场景 | 销售设备并负责安装 | 既销售货物又提供咨询服务 |

**差额征税适用情形**：

| 行业/业务 | 差额基数 | 适用税率 |
|----------|---------|---------|
| 旅游服务 | 全部价款 - 住宿/餐饮/交通/门票 | 6% |
| 劳务派遣（差额） | 全部价款 - 代发工资/社保/公积金 | 5% |
| 经纪代理 | 全部价款 - 代收代付 | 6% |
| 融资租赁 | 全部价款 - 借款利息/债券利息/车辆购置税 | 按标的适用税率 |

### 第三步：进项税额核查

**可抵扣进项**：
- 增值税专用发票
- 海关进口增值税专用缴款书
- 农产品收购/销售发票（按9%计算抵扣；用于深加工的加计1%按10%抵扣）
- 通行费电子发票
- 旅客运输服务（航空：(票价+燃油附加费)/(1+9%)x9%；铁路：票面金额/(1+9%)x9%；公路水路：票面金额/(1+3%)x3%）

**不可抵扣进项**：
- 用于简易计税/免税/集体福利/个人消费的购进
- 非正常损失对应的进项
- 购进贷款服务、餐饮服务、居民日常服务、娱乐服务

**进项转出常见情形**：

| 情形 | 转出公式 | 举例 |
|------|---------|------|
| 用于免税项目 | 已抵扣进项 x 免税收入占比 | 出口免税商品领用国内采购材料 |
| 用于集体福利 | 已抵扣进项全额转出 | 采购商品用于员工食堂 |
| 非正常损失 | 已抵扣进项 x 损失占比 | 管理不善导致存货报废 |
| 简易计税项目 | 已抵扣进项 x 简易计税收入占比 | 老项目选择简易计税 |

**简易计税适用情形速查**：

| 行业/情形 | 征收率 | 选择条件 |
|----------|--------|---------|
| 建筑业老项目（开工日期在2016.4.30前） | 3% | 可选择，选定36个月内不得变更 |
| 销售自建不动产（老项目） | 5% | 可选择 |
| 小规模纳税人销售货物 | 3%（优惠期1%） | 强制适用 |
| 一般纳税人销售自己使用过的固定资产（未抵扣进项） | 3%减按2% | 强制适用 |
| 清包工/甲供工程 | 3% | 可选择 |
| 人力资源外包 | 5% | 可选择 |

### 第四步：税负测算与预警

**应纳税额** = 销项税额 - 进项税额 - 上期留抵税额
**增值税税负率** = 应纳税额 / 不含税销售额 x 100%

**行业税负率预警**：

| 行业 | 参考税负率 | 偏低信号 | 偏高信号 |
|------|----------|---------|---------|
| 制造业 | 2-4% | 进项虚增嫌疑 | 可能遗漏进项抵扣 |
| 批发零售 | 0.5-1.5% | 进项虚增嫌疑 | 进项取票不足 |
| 建筑业 | 2-3% | 劳务外包未取票 | 材料商无票采购 |
| 房地产 | 3-5% | 土地成本抵扣异常 | 项目结算延迟 |
| 服务业 | 3-5% | 成本结构异常 | 可抵扣进项少 |
| 软件/信息技术 | 3-6% | 进项来源可疑 | 人力成本占比高无票 |

实际税负率偏离行业均值 >30% 时标注预警。

**留抵退税条件速查**（一般纳税人）：
- 增量留抵：连续6个月增量留抵 > 0，且第6个月 >= 50万元
- 纳税信用等级 A 级或 B 级
- 近36个月未骗取留抵退税/出口退税/虚开发票
- 退税额 = 增量留抵税额 x 进项构成比例 x 60%（先进制造业/集成电路等100%）

### 第五步：特殊政策适用

**出口退税（免抵退方法）**：
```
免抵退税额 = 出口离岸价 x 退税率
当期应退税额 = min(当期免抵退税额, 当期期末留抵税额)
免抵税额 = 免抵退税额 - 应退税额
不得免征和抵扣税额 = 出口离岸价 x (征税率 - 退税率)
```

**加计抵减政策**：

| 行业 | 加计比例 | 适用条件 |
|------|---------|---------|
| 生产性服务业 | 5% | 邮政/电信/现代服务/生活服务收入占比>50% |
| 生活性服务业 | 10% | 生活服务收入占比>50% |
| 集成电路设计/生产 | 15% | 符合条件的集成电路企业 |

加计抵减额 = 当期可抵扣进项税额 x 加计比例

**即征即退（软件产品）**：
```
软件产品实际税负 = (销项税额 - 进项税额) / 软件产品销售额
超过3%部分即征即退
退税额 = 实际缴纳增值税 - 软件产品销售额 x 3%
```

### 第六步：异常检查与申报底稿

**异常检查清单**：

| 检查项 | 预警信号 | 风险等级 |
|--------|---------|---------|
| 进销品类匹配 | 进项品类与主营业务不匹配 | 高 |
| 三流一致 | 发票流/资金流/货物流不一致 | 高 |
| 供应商集中度 | 单一供应商进项占比 >50% | 中 |
| 期末集中开票 | 最后5个工作日开票量 >50% | 中 |
| 顶额开票 | 金额正好卡在审批阈值 | 低 |
| 大额发票 | 单张金额超行业常规 | 中 |
| 作废率 | 发票作废率超5% | 中 |

**申报底稿**：按申报表格式整理数据——
- 一般纳税人：增值税纳税申报表主表 + 附表一（销项）+ 附表二（进项）+ 加计抵减声明（如适用）
- 小规模纳税人：小规模纳税人申报表

## 质量标准

- 税率适用须准确，混合销售和兼营须正确区分
- 进项抵扣规则须严格按增值税法规执行
- 不可抵扣进项清单须完整排查
- 进项转出情形须逐一核查
- 税负率行业参考须合理，偏离预警须量化
- 出口退税公式须正确区分免抵退和免退两种方法
- 加计抵减须验证行业收入占比条件

## 红线规则

1. **绝不建议虚开发票或买票抵扣**：任何增加进项的建议必须基于真实业务，违反即涉嫌犯罪
2. **绝不忽视进项转出**：存在免税/简易计税/福利领用等情形时，必须检查并计算进项转出金额
3. **绝不在税率适用模糊时自行判定**：混合销售/兼营行为的税率判定需用户确认业务实质后再给出结论
4. **绝不忽略优惠政策的有效期**：阶段性减免政策须标注政策到期时间，过期政策不得引用
