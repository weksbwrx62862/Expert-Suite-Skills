# /尽调清单 -- 尽职调查清单生成

> 如遇到不熟悉的占位符或需检查已连接工具，参见 [CONNECTORS.md](../../CONNECTORS.md)。

**独立能力（无需连接器）**

- 四大维度标准清单（财务/法律/业务/技术）+ 行业专项 + 架构专项
- 按交易类型差异化生成，每项标注优先级(P0/P1/P2)和负责方

**增强能力（连接器加持）**

| Connector | 增强能力 |
|-----------|---------|
| **Notion** | 尽调清单写入 Notion 项目知识库，追踪完成进度 |

> 没有连接器也完全可以使用——以上功能会通过手动输入来替代。

## 调用方式

```
/尽调清单 <项目描述或行业>
```

为这个项目生成尽调清单：@$1

## 输入要求

确认：
1. **项目名称和行业**
2. **交易类型**：少数股权投资/控股收购/并购
3. **目标公司阶段**：初创期/成长期/成熟期
4. **架构类型**：境内架构/VIE架构/红筹架构
5. **特别关注**（如有）

## 执行流程

### 第一步：企业信息预查

相关项标注"[请提供最新数据]"，用户需手动提供工商信息、股权穿透和司法风险等数据。

### 第二步：基础清单生成

**交易类型差异**：
- 少数股权 --> 侧重财务真实性+退出保障
- 控股收购 --> 增加运营细节+管理层评估+整合风险
- 并购 --> 增加反垄断+协同效应+人员整合

**一、财务尽调**

| # | 优先级 | 检查事项 | 所需材料 | 关键关注点 |
|---|-------|---------|---------|-----------|
| 1.1 | P0 | 历史财务 | 近3年1期审计报告+管报+科目余额表 | 审计意见类型、会计政策变更 |
| 1.2 | P0 | 收入核查 | 前十大客户合同+回款流水+收入确认政策 | 收入确认是否激进、关联方销售占比 |
| 1.3 | P0 | 成本核查 | 成本构成分析+供应商合同+付款记录 | 毛利率异常波动、关联方采购 |
| 1.4 | P1 | 费用核查 | 期间费用明细+研发vs管理费划分 | 研发资本化比例、股份支付 |
| 1.5 | P1 | 资产核查 | 固定资产/无形资产清单+权属证明 | 资产减值迹象、存货跌价 |
| 1.6 | P0 | 负债核查 | 有息负债明细+或有负债+担保清单 | 表外负债、员工薪酬欠缴 |
| 1.7 | P1 | 税务核查 | 各税种缴纳+税收优惠资质+涉税风险 | 高新资质真实性、转让定价 |
| 1.8 | P0 | 现金流 | 经营现金流vs净利润匹配+资金流向 | 经营CF/净利润<0.5需重点排查 |

**二、法律尽调**

| # | 优先级 | 检查事项 | 所需材料 | 关键关注点 |
|---|-------|---------|---------|-----------|
| 2.1 | P0 | 主体资格 | 营业执照+章程+历次工商变更 | 经营范围是否覆盖主营业务 |
| 2.2 | P0 | 股权结构 | 股权图+增资/转让协议+代持协议 | 代持清理、出资瑕疵、股权质押 |
| 2.3 | P0 | 合规运营 | 行业资质+数据合规(个保法)+等保 | 数据跨境、用户隐私协议 |
| 2.4 | P1 | 知识产权 | 专利/商标/软著清单+权属证明 | IP归属公司vs个人、开源合规 |
| 2.5 | P0 | 劳动人事 | 劳动合同签订率+社保公积金缴纳 | 社保差额（中国PE高频踩雷） |
| 2.6 | P0 | 诉讼仲裁 | 未决/已决诉讼+行政处罚 | 金额占净资产比例 |
| 2.7 | P1 | 重大合同 | 前十大客户/供应商合同+期限 | 合同可终止性、竞业约束 |
| 2.8 | P0 | 历史对赌 | 历史投资人对赌/回购条款 | 九民纪要下的可执行性 |

**三、业务尽调**

| # | 优先级 | 检查事项 | 关键关注点 |
|---|-------|---------|-----------|
| 3.1 | P0 | 行业分析 | TAM/增速/政策/竞争格局 |
| 3.2 | P0 | 商业模式 | 盈利模式可持续性、定价权 |
| 3.3 | P0 | 客户分析 | 集中度(前5>50%需关注)、留存率、LTV/CAC |
| 3.4 | P1 | 供应链 | 单一供应商依赖、替代方案 |
| 3.5 | P0 | 核心壁垒 | 壁垒量化评估、竞对追赶时间 |
| 3.6 | P1 | 运营指标 | GMV/MRR/ARR/DAU/留存（按行业适用） |

**四、技术尽调**（如适用）

| # | 优先级 | 检查事项 | 关键关注点 |
|---|-------|---------|-----------|
| 4.1 | P1 | 技术架构 | 可扩展性、技术债务 |
| 4.2 | P1 | 研发能力 | 研发团队规模/投入、专利产出 |
| 4.3 | P0 | 数据安全 | 等保合规、数据脱敏、灾备 |
| 4.4 | P1 | 代码资产 | 代码归属、开源组件许可合规 |

### 第三步：行业专项

| 行业 | 专项事项 |
|------|---------|
| 互联网/SaaS | 个保法/GDPR合规、云服务依赖度、开源许可、SaaS指标(NDR/GRR) |
| 医疗健康 | 药品/器械注册证、GMP认证、临床进展、医保集采影响、一致性评价 |
| 新能源/制造 | 产能利用率、环保(排污/碳排)、安全生产、土地使用权、设备折旧 |
| 消费品牌 | 商标保护、加盟合规、食品安全资质、广告法合规、退货率 |
| 金融科技 | 金融牌照(支付/小贷/担保)、利率合规(36%红线)、催收合规 |
| AI/大模型 | 训练数据版权、算法备案(深度合成)、生成式AI管理规定、算力依赖 |
| 半导体/硬科技 | 出口管制(EAR/实体清单)、设备进口依赖、政府补贴持续性、军工资质 |
| 跨境电商 | 海关合规、外汇管理、目的国准入、物流仓储、VAT合规 |
| 教育 | 办学许可、"双减"合规、预收学费监管、教师资质、ICP备案 |

### 第四步：架构专项

| 架构 | 专项事项 |
|------|---------|
| VIE | VIE协议有效性(独家咨询/股权质押/独家期权/委托投票)、外商投资负面清单、37号文登记 |
| 红筹 | 离岸公司年审、ODI备案、外汇登记、控制链完整性 |
| 境内 | 确认是否需搭建VIE/红筹（如有境外上市计划） |

### 第五步：输出交付

**如已连接 Notion：**
- 清单写入 Notion 项目知识库，设置状态/优先级/负责方字段

**如未连接：**
- Markdown表格输出，建议用户导出Excel追踪

**参考尽调时间表**：

| 阶段 | 时间 | 重点 |
|------|------|------|
| 启动（第1周） | 发材料清单+分工确认 | 商务条款锁定前启动 |
| 材料收集（第2-3周） | 催收+初审 | 重点跟催P0缺失项 |
| 现场尽调（第3-4周） | 走访+访谈 | 管理层+核心客户+供应商 |
| 报告撰写（第5周） | 汇总+确认 | 问题清单反馈给目标公司 |

## 质量自检

- [ ] 四大维度均已覆盖
- [ ] 行业专项与实际行业匹配
- [ ] 架构专项区分境内/VIE/红筹
- [ ] 每项材料具体（禁止"相关材料"等模糊表述）
- [ ] 负责方合理（财务→审计师，法律→律所）
- [ ] 九民纪要相关事项（对赌/担保/回购）已纳入
- [ ] 社保公积金合规已纳入（P0）

## 关联技能

- **筛项目** -- 尽调前先用 `/筛项目` 确认值得深入
- **审条款** -- 尽调同步用 `/审条款` 审TS/SPA
- **投决备忘录** -- 尽调完成后用 `/投决备忘录` 汇总
