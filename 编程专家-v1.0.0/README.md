# 编程专家

覆盖软件开发全流程：代码审查、代码生成、Bug诊断、重构建议、技术选型、API设计、测试用例生成、文档生成。基于Andrej Karpathy编码哲学与Harness工程化框架，倡导简洁优先、精准修改、编码前思考、目标驱动执行、Spec驱动开发、任务拆解执行、项目记忆沉淀。

> **免责声明：** 本插件辅助编程工作流程，生成的代码应经过测试和人工审核后方可用于生产环境。

## Karpathy编码哲学

本套件融入Andrej Karpathy提出的4条核心编码原则，作为所有Skill的统一行为准则：

1. **编码前思考（Think Before Coding）** — 不要假设，暴露困惑，明确权衡
2. **简洁优先（Simplicity First）** — 最少代码解决问题，不做推测性设计
3. **精准修改（Surgical Changes）** — 只碰必须碰的，保持原有风格
4. **目标驱动执行（Goal-Driven Execution）** — 定义成功标准，循环直到验证

用 `/Karpathy编码规范` 深入了解这些原则及实战案例。

## Harness工程化框架

本套件融入Harness工程化思想，将AI编程从"能写"提升到"能稳定交付"：

| 层级 | Skill | 核心能力 | 解决的问题 |
|------|-------|---------|-----------|
| **规范层** | Spec驱动开发 | Delta spec、Artifact flow | 需求漂移、AI跑偏 |
| **执行层** | 任务拆解与执行 | Wave执行、上下文隔离 | Context rot、复杂任务失真 |
| **记忆层** | 项目记忆管理 | Session summary、渐进式披露 | 跨会话失忆、规范遗忘 |
| **技能层** | 8个核心Skill | 代码生成、审查、诊断等 | 具体编程任务 |

Harness核心理念：**Agree before you build**（先对齐，再动手）、**Context engineering**（上下文工程）、**Verification closure**（验证闭环）。

## Skill 工作流

### Harness增强开发流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│ Spec驱动开发 │ → │ 任务拆解与执行│ → │   代码生成   │
│  (对齐需求)  │    │ (Wave执行)   │    │ (按Spec实现) │
└─────────────┘    └─────────────┘    └──────┬──────┘
       ↑                                      │
       │    ┌─────────────────────────────────┘
       │    │
       │    ▼
       │  ┌──────────┐   ┌──────────┐   ┌──────────┐
       │  │ 代码审查  │ ← │ 测试用例  │ ← │ 重构建议  │
       │  │(Spec检查) │   │ (Spec覆盖)│   │(Delta Spec)│
       │  └────┬─────┘   └──────────┘   └──────────┘
       │       │
       │  ┌────┴────┐   ┌──────────┐
       └──┤ Bug诊断  │ ← │ 文档生成  │
          │(记忆更新)│   │(Spec同步) │
          └─────────┘   └──────────┘
```

### 典型开发流程

1. **需求对齐** → `/Spec驱动开发` 将模糊需求转化为可验收的spec
2. **任务规划** → `/任务拆解与执行` 将spec拆分为Wave执行计划
3. **编码前** → `/Karpathy编码规范` 理解编码哲学，确保后续Skill行为一致
4. 新项目启动 → `/技术选型` 确定技术栈（决策自动记录）
5. 接口定义 → `/API设计` 设计接口规范（输出Spec Artifact）
6. 编码实现 → `/代码生成` 生成核心代码（按spec验收，遵循简洁优先）
7. 自测保障 → `/测试用例生成` 生成单元测试（覆盖spec scenario）
8. 代码审查 → `/代码审查` 发现潜在问题（含Karpathy维度+Spec一致性）
9. 持续优化 → `/重构建议` 优化代码结构（输出Delta Spec）
10. 问题排查 → `/Bug诊断` 定位并修复Bug（更新项目记忆）
11. 文档沉淀 → `/文档生成` 生成技术文档（与spec同步）
12. **记忆更新** → `/项目记忆管理` 记录session summary和决策
