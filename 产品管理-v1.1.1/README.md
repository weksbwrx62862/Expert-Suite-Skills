# 产品管理

<!-- [优化:场景] 重写README，增加能力层级图、更详细的Skill说明和MCP增强信息 -->

产品经理全流程工具箱，覆盖PRD撰写、用户故事拆解、竞品分析、需求优先级排序、Release Notes生成和用户反馈分析六大核心场景。每个技能内置行业方法论、条件分支和输出模板，产出可直接交付。

> **免责声明：** 本插件辅助专业工作流程，不替代专业意见。所有输出应由具备相应资质的专业人士审核后方可用于决策。

## How It Works

```
┌────────────────────────────────────────────────┐
│  ALWAYS（独立工作）                              │
│  ✓ PRD撰写（按产品类型差异化模板）               │
│  ✓ 用户故事拆解（INVEST校验+5种拆分模式）        │
│  ✓ 竞品分析（功能矩阵+SWOT+差异化策略）          │
│  ✓ 需求排序（RICE/ICE/MoSCoW/Kano四框架）       │
│  ✓ Release Notes（多渠道适配+内部changelog）      │
│  ✓ 用户反馈分析（分类+情感+痛点排序）            │
├────────────────────────────────────────────────┤
│  SUPERCHARGED（连接工具后增强）                    │
│  + Notion → PRD直接写入知识库                     │
│  + WebSearch → 竞品实时信息获取                   │
└────────────────────────────────────────────────┘
```

## 适用角色

- **产品经理** — PRD撰写、需求拆解、优先级排序的核心工具
- **产品总监** — 竞品分析、产品战略、反馈洞察
- **项目经理** — 需求优先级排序、版本管理、Sprint规划
- **运营经理** — 用户反馈分析、Release Notes撰写

## 快捷命令

| 命令 | 说明 | 典型输入 |
|------|------|---------|
| `/PRD生成` | 生成结构化PRD | "做一个会员积分商城" |
| `/用户故事拆解` | Epic拆解为User Story | "用户可以微信支付购买会员" |
| `/竞品分析` | 竞品多维度对比分析 | "分析飞书、钉钉、企业微信" |
| `/需求优先级排序` | 需求优先级排序 | "1.评价功能 2.推荐 3.导出" |
| `/版本Release Notes` | 版本更新说明 | "v2.3新增积分功能、修复闪退" |
| `/用户反馈分析` | 用户反馈洞察报告 | 上传反馈Excel |

## Skills

| Skill | 说明 | 核心方法论 |
|-------|------|----------|
| PRD生成 | 按产品类型生成差异化PRD | 产品类型分支 + 功能拆解三步法 + 质量自检清单 |
| 用户故事拆解 | Epic拆解为User Story | INVEST校验 + 5种拆分模式 + Story Point估算 |
| 竞品分析 | 竞品多维度对比分析 | 功能矩阵 + SWOT + 差异化三层策略 |
| 需求优先级排序 | 需求优先级排序 | RICE/ICE/MoSCoW/Kano + 框架选择决策树 |
| 版本Release Notes | 多渠道版本更新说明 | 变更分类决策表 + 渠道适配模板 |
| 用户反馈分析 | 用户反馈分类与洞察 | 6大类分类体系 + 痛点优先级模型 |
