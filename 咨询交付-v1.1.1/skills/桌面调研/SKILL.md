# /桌面调研 -- 结构化桌面研究报告

**独立能力（无需连接器）**

- 假设驱动的结构化调研框架
- 信息源可信度 Tier1-5 分级体系
- 市场规模 Top-down / Bottom-up 双法交叉验证
- PESTLE 六维全扫描 + 波特五力
- 三档深度（快速扫描/标准/深度）自适应

**增强能力（连接 ~~Notion 后）**

- 将调研报告自动发布至 Notion
- 团队可在 Notion 上协作补充数据和评审

## 连接器（可选增强）

| 连接器 | 增强能力 |
|--------|---------|
| **~~Notion** | 将调研报告自动发布至 Notion，方便团队协作和数据更新 |

> 没有连接器也完全可以使用——报告以 Markdown 格式直接输出，用户可手动复制到目标平台。

以假设驱动方式快速构建结构化调研报告。支持快速扫描（2000字）、标准调研（5000字）和深度调研（8000字+多框架）三种模式。**所有数据点必须标注来源或标注[需验证]**，引导用户用权威渠道补充。本技能内嵌信息源可信度分级、市场规模测算双验证方法论和PESTLE全维度必查清单，确保输出达到专业咨询级别。

## 调用方式

```
/桌面调研 <研究主题+调研目的>
```

## 输入要求

1. **研究主题**：行业/市场/技术/公司/政策——按MECE原则切分范围
2. **调研目的**：战略决策/投资判断/业务规划/竞品分析/可研评估/市场进入
3. **深度要求**：快速扫描（2小时出2000字）/ 标准调研（1-2天出5000字）/ 深度调研（3-5天出8000字+多框架）
4. **特别关注**：需重点回答的假设或关键问题
5. **已有材料**：可上传已有报告/数据/新闻剪报
6. **行业成熟度**（如已知）：成熟行业（侧重竞争格局）/ 新兴行业（侧重技术路径和政策走向）

> 分支判断——深度选择：
> - 仅需了解概貌、为内部讨论做准备 → **快速扫描**
> - 为项目立项、客户提案提供背景支撑 → **标准调研**
> - 为战略决策、投资决策提供系统化证据 → **深度调研**
> - 如用户未明确深度，默认按**标准调研**执行，并在输出开头提示用户可升级为深度调研

## 执行流程

### 第一步：界定范围与初始假设

按金字塔原理自顶向下：明确研究边界（地域/时间/行业细分），提出2-3个初始假设，构建Issue Tree MECE拆分子问题，确定关键知识缺口。

> 注意：范围界定是调研质量的第一道防线。常见错误包括：范围过大导致蜻蜓点水（如"中国消费市场"应缩窄到"中国Z世代线上美妆消费"）、范围过小导致缺乏战略视野（如只看单一SKU不看品类趋势）。

**行业成熟度决策分支**：
- **成熟行业**（如汽车、银行、零售）：侧重竞争格局分析（CR3/CR5/战略群组）、效率提升空间、并购整合趋势、细分市场渗透率差异
- **新兴行业**（如AI大模型、合成生物、量子计算）：侧重技术成熟度曲线（Gartner Hype Cycle位置）、政策走向与监管态度、资本投入趋势、产业链成熟度、商业模式验证阶段
- **转型中行业**（如传统零售数字化、能源转型）：侧重转型驱动力、先行者案例、转型路径选择、转型投入与回报周期

### 第二步：多源信息搜集（按可信度分级）

使用 WebSearch 按 Tier1→Tier2→Tier3→Tier4 优先级搜索，自动交叉验证多个来源的数据一致性，对搜索结果去重、按时效排序、标注数据年份。用户提供的材料同步整合分析。对于无法通过搜索获取的数据，标注[需验证]并列出建议查证的具体数据源。

按以下**信息源可信度分级表**确定搜索优先级和引用权重：

| 层级 | 来源类型 | 典型来源 | 可信度 | 引用规则 |
|------|---------|---------|--------|---------|
| **Tier 1** | 政府/监管机构 | 国家统计局(data.stats.gov.cn)、央行、各部委年报、行业协会官方数据 | 最高 | 可直接引用，标注发布日期 |
| **Tier 2** | 头部咨询公司 | McKinsey、BCG、Bain、Roland Berger、德勤、普华永道研报 | 高 | 可直接引用，标注报告名+年份 |
| **Tier 3** | 国内研究机构 | 艾瑞咨询、前瞻产业研究院、IDC中国、赛迪研究院、中金研究 | 中高 | 可引用，建议与Tier1/Tier2交叉验证 |
| **Tier 4** | 媒体报道 | 36氪、财新、经济观察报、界面新闻、行业垂直媒体 | 中 | 需交叉验证，标注"据XX报道[需验证]" |
| **Tier 5** | 自媒体/论坛 | 微信公众号、知乎、雪球、脉脉、行业论坛 | 低 | 仅做定性参考，不可作为核心论据，标注"社交媒体信息[仅供参考]" |

**国内核心数据源速查表**：

| 数据类型 | 推荐数据源 | 网址/入口 | 典型用途 |
|---------|-----------|----------|---------|
| 宏观经济 | 国家统计局 | data.stats.gov.cn | GDP、CPI、行业产值、人口 |
| 学术研究 | 中国知网(CNKI) | cnki.net | 行业综述、技术趋势、学位论文 |
| 工商信息 | 天眼查/企查查 | tianyancha.com / qcc.com | 企业股权、融资、诉讼、关联关系 |
| 上市公司 | 巨潮资讯网 | cninfo.com.cn | 年报、招股书、公告、财务数据 |
| 互联网数据 | 艾瑞咨询 | iresearch.cn | 互联网/消费/科技行业数据 |
| 产业研究 | 前瞻产业研究院 | qianzhan.com | 产业链图谱、市场规模预测 |
| 国际数据 | Bloomberg / Capital IQ / Statista | - | 全球行业数据、跨国对比 |
| 政策法规 | 国务院政策文件库 / 各部委官网 | gov.cn | 产业政策、补贴、准入门槛 |
| 专利技术 | 国家知识产权局 / Google Patents | cnipa.gov.cn | 专利布局、技术路线图 |

> 注意：数据时效性是常见陷阱。所有引用数据必须标注年份，超过2年的数据须标注"[数据年份：20XX，建议更新]"。不同来源的同一指标差异>30%时须特别说明并分析差异原因。

### 第三步：框架分析

根据调研深度级别选择分析框架：

**快速扫描模式**（2000字）：
- 市场概况（规模/增速/阶段）
- 竞争格局（主要玩家/份额）
- 关键趋势（3-5条）

**标准调研模式**（5000字）：
- 市场概况 + 市场规模测算
- 竞争格局（CR3/CR5/战略群组）
- PESTLE六维扫描（精简版）
- 关键趋势与风险
- 知识缺口与建议

**深度调研模式**（8000字+）：
- PESTLE六维全扫描
- 波特五力模型
- 产业链图谱（上→中→下游附加值分布）
- 市场规模测算（双方法交叉验证）
- 竞争格局（CR3/CR5/战略群组矩阵/竞争维度分析）
- 技术路线图（如适用）
- 假设验证汇总

**PESTLE各维度必查清单**：

| 维度 | 必查要素 | 核心问题 |
|------|---------|---------|
| **P**olitical 政治 | 产业政策、准入门槛、补贴退坡、监管趋势、中美关系影响 | 政策方向对行业是利好还是利空？监管收紧还是放松？ |
| **E**conomic 经济 | GDP增速、CPI趋势、汇率、利率环境、消费者购买力、投融资热度 | 宏观经济是否支撑行业增长？融资环境冷暖？ |
| **S**ocial 社会 | 人口结构、城镇化率、消费习惯变迁、老龄化、Z世代偏好 | 需求端人群结构如何变化？消费行为有何新趋势？ |
| **T**echnological 技术 | 技术成熟度曲线位置、关键专利格局、研发投入趋势、技术替代风险 | 核心技术处于什么阶段？是否有颠覆性技术风险？ |
| **L**egal 法律 | 行业准入法规、合规要求、知识产权保护、数据安全法、反垄断 | 法规是否构成进入壁垒？合规成本多高？ |
| **E**nvironmental 环境 | 双碳目标、ESG要求、环保合规、绿色供应链、碳交易 | 环保政策是否改变行业成本结构或竞争格局？ |

**市场规模测算——双方法交叉验证**：

市场规模测算必须采用两种方法并交叉验证：

**方法一：自上而下（Top-down）**——宏观切入
- 公式：TAM(Total Addressable Market) x 目标渗透率 x 目标市场份额
- 数据来源：Tier1-Tier2级数据源
- 优点：逻辑清晰、数据可得性高
- 缺点：假设敏感度高，渗透率估计容易偏差

**方法二：自下而上（Bottom-up）**——微观验证
- 公式：目标客户数 x 客单价 x 购买频次
- 数据来源：企业调研、专家访谈、行业报告
- 优点：贴近真实业务逻辑
- 缺点：数据获取难度大、样本代表性风险

> 验证规则：两种方法测算结果偏差<20%为可信；偏差在20%-50%需分析差异原因并说明哪组假设更可靠；偏差>50%须标注[测算存在较大不确定性]并建议通过专家访谈或一手数据补充验证。

### 第四步：洞察提炼与假设验证

逐一验证初始假设（证实/证伪/部分证实），每个关键发现回答**So What**，标注意外发现和新假设，列出数据缺口及建议补充路径（哪个数据库/哪类专家/哪份报告）。

> 注意：常见错误是只罗列信息不做判断。调研报告的价值不在于信息量，而在于对信息的解读和判断。每个章节结尾必须有"So What"总结段落。

### 第五步：输出与发布

**如果连接了 ~~Notion：**
1. 将调研报告自动发布至 Notion
2. 保持报告结构和表格格式
3. 返回文档链接，方便团队协作补充和审阅

**如果未连接：**
1. 以 Markdown 格式直接输出完整调研报告
2. 用户可手动复制到目标文档平台

## 质量标准

- **来源标注**：每个定量数据点必须标注来源或[需验证]，绝不凭空编造数据
- **可信度分级**：来源须按Tier1-Tier5分级标注，核心结论不可仅依赖Tier4-Tier5来源
- **MECE原则**：研究范围和子问题拆分不重不漏
- **假设驱动**：围绕假设验证展开，而非漫无目的罗列信息
- **So What检验**：每个关键发现必须回答"这意味着什么"
- **时效标注**：所有数据标注年份，超过2年须提示时效风险
- **双法验证**：市场规模测算须用自上而下和自下而上两种方法交叉验证
- **可操作性**：知识缺口须给出具体补充路径建议

## 红线规则

1. **绝不编造数据**：任何定量数据必须有明确来源，无法确认的标注[需验证]，严禁杜撰市场规模、增速、份额等数字
2. **绝不伪造来源**：不可编造不存在的研究报告、机构名称或报告编号
3. **Tier5来源不可作为核心论据**：社交媒体、论坛信息仅供定性参考，不可支撑关键结论
4. **不做预测性承诺**：市场规模测算是"基于假设的估算"，不可表述为"确定性预测"
5. **不隐藏不确定性**：数据缺口、假设薄弱点、信息矛盾必须如实披露
6. **不抄袭**：引用研究机构观点须标注来源，不可将他人结论当作自己的分析
7. **超出能力范围须转介**：涉及法律合规、税务、技术专利深度分析等专业领域，须建议用户咨询专业人士

## 灰色地带处理

1. **不同来源数据矛盾**：当Tier1-3级来源的同一指标差异>30%时，并列展示各来源数据，分析差异可能原因（统计口径不同、时间窗口不同、样本差异），给出分析师判断并标注"[分析师判断，建议验证]"
2. **行业数据严重滞后**：某些细分行业（如部分B2B领域）公开数据可能滞后3-5年。此时应标注数据年份，结合专家访谈或企业调研进行外推估算，并明确标注估算方法和置信度
3. **新兴行业无历史数据**：对于极早期行业（如脑机接口、室温超导应用），应以技术路线图、专利分析、资本投入趋势等替代传统市场规模测算，并标注"[市场处于早期，规模测算参考价值有限]"
4. **涉及敏感行业/政策**：涉及国防、烟草、博彩等行业或政治敏感话题时，仅引用官方公开信息，不做政策预判，建议用户咨询行业专家或法律顾问
5. **客户要求的调研范围过大**：如用户要求"调研整个消费行业"，应主动建议缩窄范围，并提供2-3个可行的缩窄方案供选择

## 输入不足处理

- 仅提供研究主题时：输出快速扫描版（2000字概览），标注"信息有限，建议补充调研目的和深度要求后获得更完整分析"
- 缺少关键信息时：主动向用户追问最重要的 2-3 个数据点（如调研目的、深度要求、特别关注领域）
- 绝不编造数据、案例或市场信息——无法验证的信息标注[需验证]
- **快速模式**：用户信息充足且明确深度 → 直接执行，不多余追问
- **引导模式**：用户仅给出模糊主题 → 先输出快速扫描，同时追问关键参数引导升级

## Agent 工具增强

- **WebSearch**：搜索行业报告、公司公开信息、政策法规等，按信息源可信度分级表优先搜索Tier1-Tier3来源，标注来源和时效
- **文件处理**：支持读取用户上传的 PDF 报告、Excel 数据、PPT 方案等，提取关键数据点整合进调研框架
- **代码执行**：市场规模测算、增长率计算、数据交叉验证等场景使用 Python 确保计算精度
- **图像识别**：可解读用户上传的图表截图、产业链图谱、组织架构图等
