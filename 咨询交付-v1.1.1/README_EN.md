# Consulting Delivery

A full-cycle management consulting toolkit covering seven core scenarios: desk research, interview notes, framework design, report writing, benchmarking, weekly status reports, and executive briefings.

> **Disclaimer:** This plugin assists professional workflows and does not replace professional advice. All outputs should be reviewed by qualified professionals before being used for decision-making.

## Target Users

- **Consultants** -- Core tool for day-to-day research and report writing
- **Project Managers** -- Project management and client reporting
- **Partners** -- Executive briefings and framework design
- **Research Analysts** -- Desk research and data analysis

## Slash Commands

| Command | Description |
|---------|-------------|
| `/desk-research` | Enter a topic to produce a structured research report (quick scan / standard / deep -- three tiers) |
| `/interview-notes` | Upload an interview transcript to produce McKinsey-standard notes (with credibility tagging) |
| `/framework-design` | Enter a client problem to produce a MECE framework skeleton (Issue Tree + Hypothesis Tree) |
| `/report-writing` | Enter analysis conclusions to produce a Pyramid Principle consulting report |
| `/benchmarking` | Enter a target and benchmark companies to produce a five-dimension benchmarking report |
| `/weekly-status-report` | Enter this week's progress to produce a seven-section structured weekly report (with RAG status) |
| `/executive-briefing` | Enter project progress to produce a one-page executive summary + backup appendix |

## Skills

| Skill | Description |
|-------|-------------|
| Desk Research | Hypothesis-driven structured research with embedded source credibility grading and dual-method market sizing |
| Interview Notes | Turn transcripts or handwritten notes into standardized notes with a four-tier credibility tagging system |
| Framework Design | MECE decomposition of a client problem into an Issue Tree, Hypothesis Tree, and Workstream plan |
| Report Writing | Pyramid Principle writing: conclusion first, top-down, TSS paragraph structure |
| Benchmarking | Five-dimension quantitative benchmarking with gap attribution analysis and a three-phase catch-up roadmap |
| Weekly Status Report | One-page principle; RAG status determination; decision items framed as options, not open-ended questions |
| Executive Briefing | McKinsey-style one-page summary with RAG status + Q&A rehearsal + executive-language standards |
