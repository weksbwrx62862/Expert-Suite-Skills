# 咨询交付

管理咨询全流程工具，覆盖桌面调研、访谈纪要、方案框架、报告撰写、标杆对比、项目周报和CEO汇报七大场景。

> **免责声明：** 本插件辅助专业工作流程，不替代专业意见。所有输出应由具备相应资质的专业人士审核后方可用于决策。

## 适用角色

- **咨询顾问** — 日常调研和报告撰写的核心工具
- **项目经理** — 项目管理和客户汇报
- **合伙人** — CEO汇报和方案框架设计
- **研究分析师** — 桌面调研和数据分析

## 快捷命令

| 命令 | 说明 |
|------|------|
| `/桌面调研` | 输入主题，输出结构化调研报告（快速扫描/标准/深度三档） |
| `/访谈纪要` | 上传访谈文字稿，输出McKinsey标准纪要（含可信度标记） |
| `/方案框架` | 输入客户问题，输出MECE方案骨架（Issue Tree + 假设树） |
| `/写报告` | 输入分析结论，输出金字塔原理结构的咨询报告正文 |
| `/标杆对比` | 输入目标与标杆公司，输出五维对标分析报告 |
| `/项目周报` | 输入本周进展，输出七板块结构化周报（含RAG状态） |
| `/CEO汇报` | 输入项目进展，输出一页纸执行摘要 + Backup附页 |

## Skills

| Skill | 说明 |
|-------|------|
| 桌面调研 | 假设驱动的结构化调研，内嵌信息源可信度分级和市场规模双法测算 |
| 访谈纪要 | 将录音转写/手写笔记整理为标准化纪要，四级可信度标记体系 |
| 方案框架 | MECE拆解客户问题，构建Issue Tree、假设树和Workstream计划 |
| 写报告 | 金字塔原理写作，结论先行、以上统下、TSS段落结构 |
| 标杆对比 | 五维度量化对标，差距归因分析和三阶段追赶路径规划 |
| 项目周报 | 一页纸原则，红黄绿灯状态判定，决策事项给选项不给开放题 |
| CEO汇报 | McKinsey风格一页纸摘要，RAG状态 + Q&A预演 + 高管语言标准 |
